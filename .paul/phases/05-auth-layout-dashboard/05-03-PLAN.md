<acceptance_criteria>

## AC-1: Stat Cards Live
```gherkin
Given user sudah login dan ada data di database
When user membuka /dashboard
Then 3 stat cards tampil: "Jumlah Siswa Aktif", "Pembayar Bulan Ini", "Penerimaan Bulan Ini"
  Dan masing-masing menampilkan angka dari GET /api/reports/dashboard
  Dan "Penerimaan Bulan Ini" diformat sebagai Rupiah (Rp xxx.xxx)
```

## AC-2: Info Yayasan + Quick Action
```gherkin
Given user membuka /dashboard
When halaman render
Then info card tampil dengan logo PNG, nama yayasan, alamat, telp, email, website
  Dan 2 quick action button tampil: "Transaksi Baru" (→ /keuangan/transaksi/baru) dan "Lihat Laporan" (→ /laporan/harian)
```

## AC-3: Tabel Riwayat Transaksi
```gherkin
Given ada transaksi di database
When user membuka /dashboard
Then tabel riwayat tampil dengan kolom: No. Transaksi, Nama Siswa, NIS, Nominal, Status, Tanggal
  Dan hanya 5 transaksi terbaru yang ditampilkan
  Dan nominal diformat Rupiah dan berwarna green (#00A651)
  Dan status "aktif" tampil sebagai badge hijau, "void" sebagai badge merah
```

## AC-4: Loading & Error State
```gherkin
Given data belum selesai di-fetch
When halaman pertama kali render
Then skeleton/spinner tampil di area stat cards dan tabel
  Dan jika API error, tampil pesan error yang tidak crash aplikasi
```

<acceptance_criteria>
## AC-1: Template Info
Given GET `/billing/payment-management/template?paymentPostId=1&schoolYearId=1`
When template untuk kombinasi itu ada
Then return `{ id, paymentPostId, schoolYearId, amount, paymentPostName, schoolYearName }`
When template tidak ada
Then return `null` (bukan 404)

## AC-2: Siswa Dengan Tagihan (Tabel 1)
Given GET `/billing/payment-management/with-bills?paymentPostId=1&schoolYearId=1`
When ada template untuk POS+Tahun tersebut
Then return array `{ studentId, studentName, nis, className, classId, billId, amount, paidAmount, status }` — siswa yang sudah punya bill untuk template itu
Filter opsional: `classId`, `unitId`, `search` (cari NIS atau nama)

## AC-3: Siswa Tanpa Tagihan (Tabel 2)
Given GET `/billing/payment-management/without-bills?paymentPostId=1&schoolYearId=1`
When ada template untuk POS+Tahun tersebut
Then return array `{ studentId, studentName, nis, className, classId }` — siswa AKTIF yang belum punya bill untuk template itu
Filter opsional: `classId`, `unitId`, `search`

## AC-4: Buat Tagihan
Given POST `/billing/payment-management/bills` dengan body `{ bills: [{ studentId, amount }], paymentPostId, schoolYearId }`
When template belum ada
Then auto-create template dengan amount default dari request pertama, lalu buat bills
When template sudah ada
Then buat bills dengan amount dari request, linked ke template.Id
Response: `{ created: number, bills: [{ id, studentId, amount }] }`

## AC-5: Edit Nominal Tagihan
Given PATCH `/billing/payment-management/bills/:id` dengan body `{ amount: 200000 }`
When bill ada DAN amount baru >= total transaksi yang sudah dibayar
Then update amount + status recalculation
When amount baru < total transaksi yang sudah dibayar
Then 400 "Nominal tidak boleh lebih kecil dari yang sudah dibayarkan (Rp X)"

## AC-6: Hapus Tagihan
Given DELETE `/billing/payment-management/bills/:id`
When bill tidak punya transaksi aktif
Then hapus bill + bill_months terkait → 200
When bill punya transaksi aktif
Then 409 "Tagihan tidak dapat dihapus karena sudah memiliki transaksi pembayaran"

## AC-7: Build & Guard
Given `pnpm --filter backend build`
When semua endpoint terimplementasi
Then TypeScript compile tanpa error. Semua endpoint dilindungi guard (`bill.view` atau permission baru).
</acceptance_criteria>

    Guard: `@Permissions('bill.view')`
    
    Catatan: paidAmount = SUM(transaction_items.amount) WHERE transaction_items.bill_id = bills.id AND parent transaction status = 'active'
  </action>

<task type="auto">
  <name>PATCH /bills/:id — edit nominal tagihan</name>
  <files>apps/backend/src/billing/payment-management/payment-management.service.ts, payment-management.controller.ts</files>
  <action>
    Logic:
    1. Cari bill by ID, 404 jika tidak ada
    2. Hitung totalPaid = SUM(transaction_items.amount) JOIN active transactions
    3. Jika amount < totalPaid → 400 dengan message "Nominal tidak boleh lebih kecil dari yang sudah dibayarkan (Rp {totalPaid})"
    4. UPDATE bills.totalAmount = amount, recalculate status:
       - amount == 0 → 'belum_bayar'
       - totalPaid >= amount → 'lunas'
       - totalPaid > 0 && totalPaid < amount → 'cicilan'
       - else 'belum_bayar'
    5. Return updated bill
    
    Controller: `@Patch('bills/:id')`, body: `{ amount }`.
    Guard: `@Permissions('bill.create')`
  </action>
  <verify>curl -X PATCH http://localhost:3000/api/billing/payment-management/bills/1 -H "Authorization: Bearer TOKEN" -H "Content-Type: application/json" -d '{"amount":200000}'</verify>
  <done>AC-5 satisfied: nominal diupdate dengan constraint paidAmount.</done>
</task>

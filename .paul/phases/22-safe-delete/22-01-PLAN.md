<acceptance_criteria>

## AC-1: Endpoint 409 dengan relatedData
```gherkin
Given sebuah record masih memiliki data terkait di database
When client mengirim DELETE /[endpoint]/:id
Then API return HTTP 409 dengan body:
  { "statusCode": 409, "message": "Tidak dapat dihapus karena masih memiliki data terkait", "relatedData": { [key]: count } }
```

## AC-2: Delete berhasil jika tidak ada dependensi
```gherkin
Given sebuah record tidak memiliki data terkait
When client mengirim DELETE /[endpoint]/:id
Then API return HTTP 200 dan record terhapus dari database
```

## AC-3: Delete transaksi hanya untuk status void
```gherkin
Given transaksi dengan status selain 'void' (aktif/lunas)
When client mengirim DELETE /transactions/:id
Then API return HTTP 409 dengan message "Transaksi aktif tidak dapat dihapus. Gunakan fitur Void terlebih dahulu."
Given transaksi dengan status 'void'
When client mengirim DELETE /transactions/:id
Then API return HTTP 200 dan transaksi + transaction_items terhapus
```

        const [tx] = await this.db

          .where(eq(transactions.id, id))
          .limit(1);

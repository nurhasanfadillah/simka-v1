<objective>
## Goal
Buat BillingEngineService dengan dua operasi: `preview` (siapa yang akan dibuatkan tagihan) dan `generate` (buat bills + bill_months massal dalam satu transaksi DB).

## Purpose
Ini adalah fitur inti Phase 02. Operator harus bisa generate tagihan untuk seluruh siswa aktif dalam satu kelas sekaligus, tanpa duplikasi, dengan nomor invoice ter-generate otomatis.

## Output
- `POST /billing/generate/preview` — returns daftar siswa eligible + count
- `POST /billing/generate` — buat bills + bill_months untuk semua siswa eligible, dalam satu DB transaction
- Invoice number format: `INV-{POSTCODE}-{ENDYEAR}-{NNNN}` (e.g., `INV-SPP-2026-0001`)
- Idempoten: siswa yang sudah punya bill untuk template ini dilewati
</objective>

<acceptance_criteria>

## AC-1: Preview mengembalikan siswa eligible tanpa mengubah data
```gherkin
Given paymentTemplateId valid ada di DB dan ada siswa aktif di kelas tersebut
When POST /billing/generate/preview dengan body {paymentTemplateId}
Then response berisi { templateInfo, eligibleStudents: [...], totalEligible: N, alreadyGenerated: M }
And tidak ada row baru di tabel bills atau bill_months
```

## AC-2: Generate membuat bills + bill_months untuk semua siswa eligible
```gherkin
Given paymentTemplateId valid dan minimal 1 siswa eligible (belum punya bill untuk template ini)
When POST /billing/generate dengan body {paymentTemplateId}
Then bills ter-insert untuk setiap siswa eligible
And bill_months ter-insert: 12 rows per bill jika tipe 'bulanan', 1 row jika tipe 'bebas'
And response berisi { generated: N, skipped: M, bills: [...invoice numbers] }
```

## AC-3: Invoice number ter-generate dengan format benar dan unik
```gherkin
Given payment post dengan code 'SPP' dan school year endYear=2026
When generate dijalankan untuk 3 siswa
Then invoice numbers: INV-SPP-2026-0001, INV-SPP-2026-0002, INV-SPP-2026-0003
And setiap invoice number unique di tabel bills
```

## AC-4: Generate idempoten — siswa yang sudah punya bill dilewati tanpa error
```gherkin
Given generate sudah dijalankan sebelumnya untuk paymentTemplateId X
When POST /billing/generate dijalankan lagi dengan paymentTemplateId X yang sama
Then response: { generated: 0, skipped: N }
And tidak ada duplikasi di tabel bills (unique constraint tidak dilanggar)
```

    5. Return: `{ templateInfo, eligibleStudents, totalEligible, alreadyGenerated: M }`

    **Method `generate(paymentTemplateId: number)`:**
    1. Load template (sama seperti preview — throw NotFoundException jika tidak ada)
    2. Get eligible students (sama seperti preview query, minus already-generated)
    3. Jika 0 eligible: return `{ generated: 0, skipped: total students count, bills: [] }`
    4. **Dalam satu DB transaction:**
       ```typescript
       return this.db.transaction(async (tx) => {
         const invoiceNumbers: string[] = [];
         const prefix = `INV-${postCode}-${endYear}`;

         // Get current max seq number for this prefix
         const existing = await tx
           .select({ invoiceNumber: bills.invoiceNumber })
           .from(bills)
           .where(like(bills.invoiceNumber, `${prefix}-%`));
         
         let seq = existing.length > 0
           ? Math.max(...existing.map(b => {
               const parts = b.invoiceNumber.split('-');
               return parseInt(parts[parts.length - 1], 10);
             })) + 1
           : 1;

         for (const student of eligibleStudents) {
           const invoiceNumber = `${prefix}-${String(seq).padStart(4, '0')}`;
           seq++;

           // Insert bill
           const totalAmount = postType === 'bulanan' ? template.amount * 12 : template.amount;
           const [bill] = await tx
             .insert(bills)
             .values({
               invoiceNumber,
               studentId: student.studentId,
               paymentTemplateId,
               schoolYearId: template.schoolYearId,
               totalAmount,
               status: 'belum_bayar',
             })
             .returning();

           // Insert bill_months
           const monthsToCreate = buildBillMonths(postType, startYear, endYear, template.amount, bill.id);
           if (monthsToCreate.length > 0) {
             await tx.insert(billMonths).values(monthsToCreate);
           }

           invoiceNumbers.push(invoiceNumber);
         }

         return {
           generated: eligibleStudents.length,
           skipped: alreadyGeneratedCount,
           bills: invoiceNumbers,
         };
       });
       ```

    **Helper `buildBillMonths(type, startYear, endYear, amount, billId)`:**
    - Jika 'bulanan': buat 12 entries untuk tahun ajaran
      - Bulan 7-12 dengan year = startYear
      - Bulan 1-6 dengan year = endYear
      - Setiap entry: { billId, month, year, amount, status: 'belum_bayar' }
    - Jika 'bebas': buat 1 entry
      - month = new Date().getMonth() + 1, year = new Date().getFullYear()
      - amount = template.amount

    **Import yang dibutuhkan:**
    ```typescript
    import { and, eq, like } from 'drizzle-orm';
    import { bills, billMonths, paymentTemplates, paymentPosts } from '../../db/schema/financial.schema';
    import { classes, schoolYears, studentClasses, students } from '../../db/schema/academic.schema';
    ```

    AVOID:
    - Jangan pakai `db.execute(sql`...`)` untuk invoice generation — gunakan Drizzle query builder
    - Jangan insert bill_months satu per satu di loop terpisah — batch dengan `.values([...array])`
    - Jangan load semua bills untuk cek existing — filter dengan `payment_template_id = X`

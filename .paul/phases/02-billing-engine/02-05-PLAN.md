## Goal
Tambah endpoint `GET /billing/tunggakan` untuk menampilkan semua tagihan yang belum lunas (status 'belum_bayar' atau 'cicilan'), dengan filter opsional classId dan schoolYearId.

## Purpose
Bendahara dan operator perlu melihat daftar siswa yang masih punya tunggakan untuk mengingatkan pembayaran. Ini adalah deliverable terakhir Phase 02.

## Output
- `GET /billing/tunggakan?classId=N&schoolYearId=N` — list bills status belum_bayar + cicilan
- Method `findTunggakan()` di BillsService

    **1. bills.service.ts — tambah method `findTunggakan()`:**

    Import `inArray` dari 'drizzle-orm'. Tambah method di bawah `findOne()`:

    ```typescript
    findTunggakan(filters: { classId?: number; schoolYearId?: number } = {}) {
      const conditions: SQL[] = [
        inArray(bills.status, ['belum_bayar', 'cicilan']),
      ];
      if (filters.classId) conditions.push(eq(paymentTemplates.classId, filters.classId));
      if (filters.schoolYearId) conditions.push(eq(bills.schoolYearId, filters.schoolYearId));

      return this.buildBaseQuery()
        .where(and(...conditions))
        .orderBy(asc(students.name));
    }
    ```

    **2. bills.controller.ts — tambah endpoint di atas `@Get(':id')`:**

    ```typescript
    @Get('tunggakan')
    @RequirePermissions('bill.view')
    findTunggakan(
      @Query('classId') classId?: string,
      @Query('schoolYearId') schoolYearId?: string,
    ) {
      return this.service.findTunggakan({
        classId: classId ? parseInt(classId, 10) : undefined,
        schoolYearId: schoolYearId ? parseInt(schoolYearId, 10) : undefined,
      });
    }
    ```

    PENTING: `@Get('tunggakan')` harus ditempatkan SEBELUM `@Get(':id')` agar NestJS tidak salah
    meng-interpret "tunggakan" sebagai parameter `:id`.

    AVOID: Jangan buat module/service baru — cukup extend BillsService dan BillsController yang ada.

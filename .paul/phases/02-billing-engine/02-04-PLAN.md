<objective>
## Goal
Buat Bills API: list tagihan dengan filter multi-kolom, dan detail satu tagihan beserta rincian bulan (bill_months).

## Purpose
Setelah generate, operator perlu melihat daftar tagihan (siapa yang sudah ditagih, berapa, status apa) dan detail per tagihan. Ini adalah read-only API — tidak ada mutasi data tagihan di plan ini (mutasi status terjadi di Phase 03 saat transaksi).

## Output
- `GET /billing/bills` — list bills dengan optional filter + enriched data (nama siswa, kelas, pos, tahun ajaran)
- `GET /billing/bills/:id` — detail bill + array bill_months
</objective>

## AC-1: GET list mengembalikan bills dengan data relasi dan optional filter
```gherkin
Given bills sudah ada di database setelah generate
When GET /billing/bills dipanggil (dengan atau tanpa query params)
Then response array berisi objek dengan: id, invoiceNumber, studentName, nis, className, schoolYearName, paymentPostName, totalAmount, status, createdAt
And filter opsional bekerja: ?studentId=N, ?classId=N, ?schoolYearId=N, ?paymentTemplateId=N, ?status=belum_bayar|cicilan|lunas
```

## AC-2: GET detail mengembalikan bill + bill_months
```gherkin
Given bill dengan id X ada di database
When GET /billing/bills/:id dipanggil
Then response berisi semua field dari AC-1 PLUS billMonths: [{ id, month, year, amount, status, paidAt }]
And jika id tidak ditemukan, response 404
```

## AC-3: Filter status bekerja dengan benar
```gherkin
Given ada bills dengan status 'belum_bayar' dan 'lunas'
When GET /billing/bills?status=belum_bayar
Then hanya bills dengan status 'belum_bayar' yang dikembalikan
```

  <action>

    Inject DRIZZLE. Import semua tabel yang dibutuhkan.

    **Method `findAll(filters)`:**
    Parameter: `{ studentId?: number, classId?: number, schoolYearId?: number, paymentTemplateId?: number, status?: 'belum_bayar' | 'cicilan' | 'lunas' }`

    Query dengan JOIN:
    ```typescript
    db.select({
      id: bills.id,
      invoiceNumber: bills.invoiceNumber,
      studentId: bills.studentId,
      studentName: students.name,
      nis: students.nis,
      classId: paymentTemplates.classId,
      className: classes.name,
      schoolYearId: bills.schoolYearId,
      schoolYearName: schoolYears.name,
      paymentTemplateId: bills.paymentTemplateId,
      paymentPostName: paymentPosts.name,
      paymentPostCode: paymentPosts.code,
      totalAmount: bills.totalAmount,
      status: bills.status,
      createdAt: bills.createdAt,
      updatedAt: bills.updatedAt,
    })
    .from(bills)
    .innerJoin(students, eq(bills.studentId, students.id))
    .innerJoin(paymentTemplates, eq(bills.paymentTemplateId, paymentTemplates.id))
    .innerJoin(paymentPosts, eq(paymentTemplates.paymentPostId, paymentPosts.id))
    .innerJoin(classes, eq(paymentTemplates.classId, classes.id))
    .innerJoin(schoolYears, eq(bills.schoolYearId, schoolYears.id))
    .where(conditions.length > 0 ? and(...conditions) : undefined)
    .orderBy(asc(students.name))
    ```

    Dynamic WHERE dengan `SQL[]` (pola yang sama seperti payment-templates.service.ts):
    ```typescript
    const conditions: SQL[] = [];
    if (filters.studentId) conditions.push(eq(bills.studentId, filters.studentId));
    if (filters.classId) conditions.push(eq(paymentTemplates.classId, filters.classId));
    if (filters.schoolYearId) conditions.push(eq(bills.schoolYearId, filters.schoolYearId));
    if (filters.paymentTemplateId) conditions.push(eq(bills.paymentTemplateId, filters.paymentTemplateId));
    if (filters.status) conditions.push(eq(bills.status, filters.status));
    ```

    **Method `findOne(id: number)`:**
    1. Query bill dengan JOIN yang sama seperti findAll WHERE `bills.id = id`, `.limit(1)`
    2. Throw NotFoundException jika tidak ada
    3. Query bill_months untuk billId tersebut:
       ```typescript
       const months = await db
         .select()
         .from(billMonths)
         .where(eq(billMonths.billId, id))
         .orderBy(asc(billMonths.year), asc(billMonths.month));
       ```
    4. Return: `{ ...bill, billMonths: months }`

    AVOID:
    - Jangan gunakan Drizzle relational query (`db.query.bills.findMany`) — gunakan `db.select()` eksplisit untuk kontrol penuh
    - Jangan eager-load bill_months di findAll — hanya di findOne (performa)

    @Get()
    @RequirePermissions('bill.view')
    findAll(
      @Query('studentId') studentId?: string,
      @Query('classId') classId?: string,
      @Query('schoolYearId') schoolYearId?: string,
      @Query('paymentTemplateId') paymentTemplateId?: string,
      @Query('status') status?: 'belum_bayar' | 'cicilan' | 'lunas',
    ) {
      return this.service.findAll({
        studentId: studentId ? parseInt(studentId, 10) : undefined,
        classId: classId ? parseInt(classId, 10) : undefined,
        schoolYearId: schoolYearId ? parseInt(schoolYearId, 10) : undefined,
        paymentTemplateId: paymentTemplateId ? parseInt(paymentTemplateId, 10) : undefined,
        status,
      });

    @Get(':id')

    findOne(@Param('id', ParseIntPipe) id: number) {

    Buat direktori `apps/backend/src/billing/` sebagai domain module baru (setara dengan `master/`).

    **1. create-payment-template.dto.ts:**
    ```typescript
    import { IsInt, IsPositive, Min } from 'class-validator';

    export class CreatePaymentTemplateDto {
      @IsInt()
      @IsPositive()
      classId!: number;

      @IsInt()
      @IsPositive()
      paymentPostId!: number;

      @IsInt()
      @IsPositive()
      schoolYearId!: number;

      @IsInt()
      @Min(1000)
      amount!: number;  // rupiah, minimum 1000
    }
    ```

    **2. update-payment-template.dto.ts:**
    ```typescript
    import { IsInt, IsOptional, Min } from 'class-validator';

    export class UpdatePaymentTemplateDto {
      @IsInt()
      @Min(1000)
      @IsOptional()
      amount?: number;
    }
    ```
    Catatan: Hanya `amount` yang bisa diupdate. classId/paymentPostId/schoolYearId tidak boleh diubah setelah dibuat (immutable identity).

    **3. payment-templates.service.ts:**
    Ikuti pola payment-posts.service.ts dengan modifikasi:
    - `findAll(classId?: number, schoolYearId?: number)` — query dengan LEFT JOIN ke classes, paymentPosts, schoolYears untuk mendapat nama-nama relasi. Support optional filter.
    - `findOne(id: number)` — sama dengan pola existing, throw NotFoundException
    - `create(dto)` — insert, catch error code '23505' → ConflictException('Template tagihan sudah ada untuk kombinasi kelas, pos, dan tahun ajaran ini')
    - `update(id, dto)` — findOne dulu, lalu update hanya field amount + updatedAt

    Untuk `findAll` dengan join, gunakan Drizzle query builder:
    ```typescript
    import { eq, asc } from 'drizzle-orm';
    import { paymentTemplates, paymentPosts } from '../../db/schema/financial.schema';
    import { classes, schoolYears } from '../../db/schema/academic.schema';

    // di findAll:
    return this.db
      .select({
        id: paymentTemplates.id,
        classId: paymentTemplates.classId,
        paymentPostId: paymentTemplates.paymentPostId,
        schoolYearId: paymentTemplates.schoolYearId,
        amount: paymentTemplates.amount,
        className: classes.name,
        paymentPostName: paymentPosts.name,
        schoolYearName: schoolYears.name,
        createdAt: paymentTemplates.createdAt,
        updatedAt: paymentTemplates.updatedAt,
      })
      .from(paymentTemplates)
      .leftJoin(classes, eq(paymentTemplates.classId, classes.id))
      .leftJoin(paymentPosts, eq(paymentTemplates.paymentPostId, paymentPosts.id))
      .leftJoin(schoolYears, eq(paymentTemplates.schoolYearId, schoolYears.id))
      .orderBy(asc(schoolYears.name), asc(classes.name), asc(paymentPosts.name));
    ```

    Untuk filter opsional: tambah kondisi `where` jika parameter tidak undefined.

    **4. payment-templates.controller.ts:**
    Route: `@Controller('billing/payment-templates')`
    Endpoints:
    - `GET /` dengan `@RequirePermissions('payment_template.view')`, terima query params `?classId=N&schoolYearId=N` (optional, ParseIntPipe dengan optional flag)
    - `GET /:id` dengan `@RequirePermissions('payment_template.view')`
    - `POST /` dengan `@RequirePermissions('payment_template.create')`, `@HttpCode(201)`
    - `PATCH /:id` dengan `@RequirePermissions('payment_template.update')`
    Tidak ada DELETE endpoint.

    **5. payment-templates.module.ts:**
    ```typescript
    @Module({
      providers: [PaymentTemplatesService],
      controllers: [PaymentTemplatesController],
    })
    export class PaymentTemplatesModule {}
    ```

    **6. billing.module.ts:**
    ```typescript
    @Module({
      imports: [PaymentTemplatesModule],
    })
    export class BillingModule {}
    ```

    AVOID:
    - Jangan gunakan `@Query() query: any` — type query params dengan DTO atau parameter langsung
    - Jangan izinkan update classId/paymentPostId/schoolYearId — immutable setelah create
    - Jangan import DatabaseModule di BillingModule — DB diinject global lewat DatabaseModule di AppModule

## Backend API
- `GET /api/master/students` — fetch all active students (client-side name filter)
- `GET /api/billing/bills?studentId=N&status=belum_bayar` — student's unpaid bills
- `GET /api/billing/bills/:id` — bill detail with billMonths (for month selection)
- `POST /api/transactions` — body: `{ studentId, items: [{ billId, billMonthId?, amount }], notes? }` → `{ id, transactionNumber, totalAmount, status }`
- `GET /api/transactions/:id/receipt` — PDF download

## Key Constraints
- Client-side student search (API tidak support name query)
- 1 transaksi bisa bayar multiple tagihan sekaligus (SPP + buku + dll)
- billMonthId opsional — untuk tagihan bebas (non-bulanan)
- Transaction amount bisa berbeda dari bill amount (partial payment)

<acceptance_criteria>

## AC-1: Cari & Pilih Siswa
```gherkin
Given user membuka /keuangan/transaksi/baru
When user mengetik di input search "Cari siswa (nama/NIS)"
Then daftar siswa ter-filter secara real-time (client-side)
  Dan setiap row menampilkan: NIS, Nama, Kelas Aktif, Unit
  Dan klik row siswa → siswa terpilih, lanjut ke step pilih tagihan
  Dan jika search tidak menghasilkan match, tampil "Tidak ada siswa ditemukan"
```

## AC-2: Pilih Tagihan & Bulan
```gherkin
Given siswa sudah terpilih
When halaman memuat tagihan siswa
Then tampil daftar tagihan yang BELUM LUNAS (status: belum_bayar/cicilan)
  Dan setiap tagihan menampilkan: Invoice#, POS, Total, Status, Checkbox untuk pilih
  Dan tagihan yang dipilih (checked) menampilkan expandable section dengan detail bulan (untuk tagihan bulanan)
  Dan setiap bulan bisa di-checkbox individual (atau "Bayar Semua" untuk pilih semua bulan)
  Dan untuk tagihan "bebas": langsung bisa input nominal tanpa pilih bulan
  Dan jika tidak ada tagihan, tampil "Siswa tidak memiliki tagihan aktif"
```

## AC-3: Input Nominal & Konfirmasi
```gherkin
Given tagihan & bulan sudah dipilih
When user klik "Lanjutkan"
Then tampil ringkasan pembayaran: nama siswa, daftar item yang dipilih dengan input nominal
  Dan nominal di-pre-fill dengan sisa tagihan (amount per month untuk bulanan, remaining untuk bebas)
  Dan user bisa mengubah nominal (partial payment)
  Dan total otomatis dihitung di bagian bawah
  Dan tombol "Konfirmasi Pembayaran" untuk melanjutkan
```

## AC-4: Simpan & Kwitansi
```gherkin
Given ringkasan sudah dikonfirmasi
When user klik "Konfirmasi Pembayaran"
Then muncul dialog konfirmasi final dengan ringkasan + input notes (opsional)
  Dan klik "Bayar" → POST /transactions → sukses
  Dan tampil halaman sukses: transaction number, total, status "aktif"
  Dan tombol "Cetak Kwitansi" → download PDF dari /transactions/:id/receipt
  Dan tombol "Transaksi Baru" → reset form ke step awal
```

## AC-5: Error Handling
```gherkin
Given user di halaman transaksi
When API error terjadi
Then banner error merah tampil (tidak crash)
  Dan jika semua tagihan sudah lunas → empty state "Semua tagihan sudah lunas"
```

    **Section 1: Cari Siswa**
    ```

    <p>Cari siswa dan pilih tagihan yang akan dibayar</p>

      <Input placeholder="Cari siswa (nama/NIS)..."

        onChange={e => setSearchQuery(e.target.value)} />

              ))}
            </tbody>
          </table>
        </div>
      )}
      {searchQuery && filteredStudents.length === 0 && (
        <p className="text-sm text-gray-400 mt-3">Tidak ada siswa ditemukan</p>
      )}
    </div>
    ```

      <div>{selectedStudent.nis} — {selectedStudent.name}</div>

    </div>
    
    <!-- Bills list -->
    {bills.map(bill => (
      <div className="bg-white rounded-xl border p-4">
        <div className="flex items-center justify-between">
          <div className="flex items-center gap-3">
            <Checkbox checked={isBillSelected(bill.id)} onCheckedChange={...} />
            <div>
              <p className="font-medium">{bill.invoiceNumber} — {bill.paymentPostName}</p>
              <p className="text-sm text-gray-500">Total: {formatRupiah(bill.totalAmount)}</p>
            </div>
          </div>
          <Badge>{STATUSES[bill.status]}</Badge>
        </div>

        {isBillSelected(bill.id) && expandedBillId === bill.id && billMonths && (
          <div className="mt-3 pl-8 border-t pt-3">
            <p className="text-sm font-medium mb-2">Pilih Bulan:</p>
            {billMonths.filter(m => m.status === 'belum_bayar').map(m => (

                <Checkbox checked={isMonthSelected(bill.id, m.id)} onCheckedChange={...} />

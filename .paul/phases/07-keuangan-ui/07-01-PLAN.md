## Backend API (existing, ready)
- `GET /api/billing/payment-templates?classId=N&schoolYearId=N` — list templates dengan joined names
- `POST /api/billing/generate/preview` — body: `{ paymentTemplateId }` → templateInfo + eligibleStudents + alreadyGenerated
- `POST /api/billing/generate` — body: `{ paymentTemplateId }` → { generated, skipped, bills[] }
- `GET /api/billing/bills?studentId&classId&schoolYearId&paymentTemplateId&status` — list dengan joined names
- `GET /api/billing/bills/:id` — detail dengan billMonths array
</context>

## AC-2: Halaman Generate Tagihan
```gherkin
Given user sudah login dan ada template di database
When user membuka /keuangan/generate
Then tampil dropdown "Pilih Template" yang menampilkan "Nama Kelas — POS (Tahun) = Rp xxx"
  Dan setelah pilih template, tombol "Preview" aktif
  Dan klik "Preview" menampilkan:
    - Informasi template: Kelas, POS, Tahun, Tipe, Nominal
    - Jumlah siswa eligible + sudah generate + akan generate
    - Daftar nama siswa yang akan digenerate
  Dan tombol "Generate Tagihan" muncul setelah preview
  Dan klik "Generate" → progress indicator → hasil: {generated} tagihan berhasil, {skipped} sudah ada
  Dan tampilkan daftar invoice number yang baru dibuat
```

## AC-3: Halaman Data Tagihan
```gherkin
Given ada tagihan di database
When user membuka /keuangan/tagihan
Then tampil tabel dengan kolom: No. Invoice, Siswa, NIS, Kelas, POS, Nominal, Status, Aksi
  Dan filter: Kelas, Tahun Ajaran, Status (3 dropdown di header)
  Dan status badge: "belum_bayar" kuning, "cicilan" biru, "lunas" hijau
  Dan nominal format Rupiah hijau
  Dan tombol "Detail" membuka modal yang menampilkan semua billMonths (bulan, nominal, status)
  Dan billMonths "lunas" badge hijau, "belum_bayar" badge abu-abu
```

## Backend API
- `GET /api/transactions?studentId&status&dateFrom&dateTo` — list with student join
- `GET /api/transactions/:id` — detail with items[]
- `POST /api/transactions/:id/void` — body: `{ voidReason }` (min 3 chars)
- `GET /api/billing/bills/tunggakan?classId&schoolYearId` — same shape as bills list
- `GET /api/master/classes` + `/api/master/school-years` — filter dropdowns

## AC-1: Riwayat — Table & Filter
```gherkin
Given ada transaksi di database
When user membuka /keuangan/riwayat
Then tabel tampil: No. Transaksi, Siswa, NIS, Nominal, Status, Tanggal, Aksi
  Dan filter: Status (aktif/void), Rentang Tanggal (dateFrom/dateTo)
  Dan nominal Rupiah hijau
  Dan status "aktif" badge hijau, "void" badge merah
  Dan sorted by tanggal terbaru (dari API)
```

## AC-2: Riwayat — Detail & Void
```gherkin
Given user di halaman riwayat
When user klik "Detail" pada transaksi
Then modal tampil: info transaksi + tabel items (tagihan, bulan, nominal)
  Dan tombol "Void" muncul (hanya jika status "aktif")
  Dan klik "Void" → dialog konfirmasi dengan input alasan (min 3 karakter)
  Dan setelah void berhasil, status berubah jadi "void", tabel refresh
  Dan transaksi yang sudah "void" tidak menampilkan tombol void
```

## AC-3: Tunggakan
```gherkin
Given ada tagihan belum lunas
When user membuka /keuangan/tunggakan
Then tabel tampil: No. Invoice, Siswa, NIS, Kelas, POS, Nominal, Status
  Dan filter: Kelas, Tahun Ajaran
  Dan status badge: "belum_bayar" kuning, "cicilan" biru
  Dan nominal Rupiah hijau
```

## AC-4: Loading & Error
```gherkin
Given user di riwayat atau tunggakan
When data sedang di-fetch
Then skeleton loading tampil
  Dan error banner merah jika gagal
  Dan empty state sesuai konteks
```

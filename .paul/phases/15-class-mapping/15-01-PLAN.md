<objective>
## Goal
Tambah 3 endpoint backend yang dibutuhkan halaman class mapping: GET daftar siswa di kelas, GET siswa tersedia untuk mapping, dan DELETE lepas siswa dari kelas.

## Purpose
Frontend mapping page butuh data dari server — siapa yang sudah ada di kelas target, siapa yang bisa dipindahkan, dan kemampuan melepas siswa dari kelas. Tanpa endpoint ini halaman frontend tidak bisa dibangun.

## Output
- `GET /master/student-classes?classId=X&schoolYearId=Y` — daftar siswa di kelas
- `GET /master/students/mapping?schoolYearId=X&excludeClassId=Y` — siswa aktif di luar kelas target
- `DELETE /master/student-classes/:id` — lepas siswa dari kelas
</objective>

<acceptance_criteria>

## AC-1: GET siswa di kelas
```gherkin
Given classId dan schoolYearId valid dikirim sebagai query param
When GET /master/student-classes?classId=1&schoolYearId=1
Then response 200 array berisi { enrollmentId, studentId, nis, name, gender } untuk semua siswa di kelas tersebut
```

## AC-2: GET siswa tersedia untuk mapping
```gherkin
Given schoolYearId dan excludeClassId valid dikirim sebagai query param
When GET /master/students/mapping?schoolYearId=1&excludeClassId=2
Then response 200 array berisi semua siswa dengan studentStatus='aktif' yang TIDAK ada di kelas excludeClassId untuk schoolYearId tersebut, setiap item menyertakan enrollmentId (null jika belum terdaftar di tahun itu), currentClassId, currentClassName
```

## AC-3: DELETE lepas siswa dari kelas
```gherkin
Given enrollmentId valid (record student_classes ada)
When DELETE /master/student-classes/:id
Then response 200, record student_classes dihapus dari DB
```

## AC-4: Validasi query params wajib
```gherkin
Given request ke GET /master/student-classes tanpa classId atau schoolYearId
When request diterima
Then response 400 Bad Request
```

  <action>
    Di service, tambah method `findByClass(classId: number, schoolYearId: number)`:
    - Query: SELECT sc.id as enrollmentId, sc.studentId, s.nis, s.name, s.gender
      FROM student_classes sc
      INNER JOIN students s ON s.id = sc.studentId
      WHERE sc.classId = classId AND sc.schoolYearId = schoolYearId
    - Return array hasil join
    - Import `students` dari schema jika belum ada

    Di controller, tambah GET endpoint dengan @Get() (sebelum param route yang ada):
    - Method: `findByClass(@Query('classId') classId?: string, @Query('schoolYearId') schoolYearId?: string)`
    - Jika salah satu query param missing → throw BadRequestException('classId dan schoolYearId wajib diisi')
    - Parse keduanya ke parseInt, panggil service.findByClass(...)
    - Permission: @RequirePermissions('student.view')
    - Import BadRequestException dari @nestjs/common
    
    AVOID: Jangan ubah method enroll atau transfer yang ada
  </action>
  <verify>
    curl "http://localhost:3000/api/master/student-classes?classId=1&schoolYearId=1" dengan JWT valid
    → response 200 array (bisa kosong jika belum ada data)
    curl "http://localhost:3000/api/master/student-classes" tanpa params → response 400
  </verify>
  <done>AC-1 dan AC-4 satisfied: GET dengan params → array, tanpa params → 400</done>

<task type="auto">
  <name>Task 2: findAvailableForMapping + GET /master/students/mapping</name>
  <files>
    apps/backend/src/master/students/students.service.ts,
    apps/backend/src/master/students/students.controller.ts
  </files>
  <action>
    Di service, tambah method `findAvailableForMapping(schoolYearId: number, excludeClassId: number)`:
    - Query: LEFT JOIN student_classes sc ON sc.studentId = students.id AND sc.schoolYearId = schoolYearId
      LEFT JOIN classes c ON c.id = sc.classId
      WHERE students.studentStatus = 'aktif'
      AND (sc.classId IS NULL OR sc.classId != excludeClassId)
    - Select fields: students.id, students.nis, students.name, students.gender,
      sc.id as enrollmentId (null jika tidak ada), sc.classId as currentClassId (null), c.name as currentClassName (null)
    - Order by students.name ASC
    - Import `ne`, `isNull`, `or` dari drizzle-orm jika belum

    Di controller, tambah GET route `/mapping` SEBELUM route `:id`:
    - Method: `findAvailableForMapping(@Query('schoolYearId') schoolYearId?: string, @Query('excludeClassId') excludeClassId?: string)`
    - Jika salah satu missing → throw BadRequestException('schoolYearId dan excludeClassId wajib diisi')
    - Permission: @RequirePermissions('student.view')
    - Route literal harus mendahului param wildcard (:id) — ini Decision #20/#27 pattern yang sama
    
    AVOID: Jangan ubah findAll, findOne, atau getClasses yang ada
  </action>
  <verify>
    curl "http://localhost:3000/api/master/students/mapping?schoolYearId=1&excludeClassId=1" dengan JWT valid
    → response 200 array siswa aktif yang tidak di kelas 1 untuk tahun ajaran 1
    Setiap item punya field enrollmentId (number|null), currentClassId (number|null), currentClassName (string|null)
  </verify>
  <done>AC-2 satisfied: students aktif di luar kelas yang dipilih, dengan info enrollment tahun berjalan</done>
</task>

<task type="auto">
  <name>Task 3: remove + DELETE /master/student-classes/:id</name>
  <files>
    apps/backend/src/master/student-classes/student-classes.service.ts,
    apps/backend/src/master/student-classes/student-classes.controller.ts
  </files>
  <action>
    Di service, tambah method `remove(id: number)`:
    - Cek enrollment exists: SELECT dari studentClasses WHERE id = id, jika tidak ada throw NotFoundException('Data enrollment tidak ditemukan')
    - DELETE FROM student_classes WHERE id = id
    - Return { success: true, id }

    Di controller, tambah @Delete(':id') endpoint:
    - Method: `remove(@Param('id', ParseIntPipe) id: number)`
    - Permission: @RequirePermissions('student.update')
    - Import Delete dari @nestjs/common
    
    Route DELETE /:id tidak konflik dengan PATCH /:id/transfer karena HTTP method berbeda.
    AVOID: Jangan gunakan soft delete (set isActive=false) — hapus record sepenuhnya agar unique constraint (studentId, schoolYearId) bisa dipakai lagi
  </action>
  <verify>
    Buat enrollment test via POST, ambil id, lalu DELETE /master/student-classes/:id
    → response 200 { success: true, id }
    GET /master/student-classes?classId=X&schoolYearId=Y → siswa tersebut tidak muncul lagi
  </verify>
  <done>AC-3 satisfied: enrollment dihapus dari DB, siswa bisa dienroll ulang ke kelas lain</done>

## Goal
Buat halaman Class Mapping di `/master/kelas/mapping` — admin bisa melihat siswa di kelas, menambah siswa (drag dari dual table), dan melepas siswa dari kelas dengan aksi Lepas/Proses.

- Tipe baru di `types/master.ts` untuk response mapping API

<acceptance_criteria>

## AC-1: Filter bar berfungsi
```gherkin
Given halaman /master/kelas/mapping dibuka
When tahun pelajaran aktif otomatis terpilih di dropdown pertama
Then dropdown Unit dan Kelas aktif, kelas-kelas terfilter sesuai unit yang dipilih
```

## AC-2: Tabel siswa kelas saat ini
```gherkin
Given tahun pelajaran, unit, dan kelas sudah dipilih
When data kelas terpilih dimuat
Then tabel atas menampilkan NIS, Nama, Jenis Kelamin siswa yang sudah ada di kelas, dengan tombol "Lepas" per baris
```

## AC-3: Aksi Lepas siswa dari kelas
```gherkin
Given tabel atas menampilkan siswa di kelas
When admin klik tombol "Lepas" pada baris siswa X
Then DELETE /master/student-classes/:enrollmentId dipanggil, siswa X hilang dari tabel atas, muncul di tabel kiri (tersedia)
```

## AC-4: Dual table — klik kiri masuk kanan
```gherkin
Given tabel kiri menampilkan siswa tersedia (tidak di kelas terpilih)
When admin klik baris siswa di tabel kiri
Then siswa tersebut pindah ke tabel kanan (staging), hilang dari tabel kiri
```

## AC-5: Aksi Batal di tabel kanan
```gherkin
Given tabel kanan (staging) berisi siswa yang akan di-assign
When admin klik "Batal" pada baris siswa di tabel kanan
Then siswa kembali ke tabel kiri, tabel kanan mengecil
```

## AC-6: Filter tabel kiri
```gherkin
Given tabel kiri menampilkan daftar siswa tersedia
When admin mengetik di filter NIS atau Nama atau memilih filter Kelas Saat Ini
Then tabel kiri difilter sesuai input tanpa reload halaman
```

## AC-7: Proses assign siswa
```gherkin
Given tabel kanan berisi 1+ siswa staging
When admin klik tombol "Proses"
Then setiap siswa di staging di-assign ke kelas target:
  - Jika enrollmentId null → POST /master/student-classes (enroll baru)
  - Jika enrollmentId ada → PATCH /master/student-classes/:id/transfer
Setelah semua selesai: tabel atas direfresh, staging dikosongkan, tabel kiri direfresh
```

  <files>apps/frontend/src/pages/master/kelas/mapping.tsx</files>
  <action>
    Buat file baru `apps/frontend/src/pages/master/kelas/mapping.tsx`.

    **Import yang dibutuhkan:**
    - useState, useEffect dari react
    - apiClient dari @/lib/api
    - SchoolYear, SchoolUnit, Class, ClassMember, MappingAvailableStudent dari @/types/master
    - shadcn: Select, Button, Input, Label, Badge (jika ada), atau buat inline styling
    - Ikuti pola import halaman kelas/index.tsx yang sudah ada

    **State:**
    ```
    schoolYears: SchoolYear[]
    schoolUnits: SchoolUnit[]
    classes: Class[]  // filtered by selected unit
    selectedYearId: number | null  // default: tahun aktif (isActive=true)
    selectedUnitId: number | null
    selectedClassId: number | null

    currentMembers: ClassMember[]        // siswa di kelas terpilih
    availableStudents: MappingAvailableStudent[]  // sumber tabel kiri
    staging: MappingAvailableStudent[]   // tabel kanan

    filterNis: string
    filterName: string
    filterClass: string  // filter kelas saat ini di tabel kiri

    loading: boolean     // untuk Proses button
    ```

    **Layout (dari atas ke bawah):**

    1. **Header bar:** judul "Mapping Kelas", subtitle "Assign siswa ke kelas"

    2. **Filter bar (3 dropdown satu baris):**
       - Select Tahun Pelajaran → load dari GET /master/school-years, auto-select yang isActive=true
       - Select Unit Sekolah → load dari GET /master/school-units
       - Select Kelas → load dari GET /master/classes?school_unit_id=X (filter setelah unit dipilih), disable jika unit belum dipilih
       - Saat kelas berubah → fetch currentMembers dan availableStudents, kosongkan staging

    3. **Tabel atas "Siswa di Kelas [nama kelas]"** (tampil hanya jika kelas dipilih):
       - Kolom: NIS, Nama, Jenis Kelamin, Aksi
       - Aksi per baris: Button "Lepas" (variant destructive/outline, size sm)
       - onClick Lepas → DELETE /master/student-classes/:enrollmentId, lalu refresh currentMembers + availableStudents
       - Tampilkan jumlah siswa di header tabel: "{n} siswa"

    4. **Divider atau separator**

    5. **Dual table (2 kolom, side by side):**
       - **Tabel kiri header:** "Siswa Tersedia ({n})"
         - 3 filter input di atas tabel: Input NIS, Input Nama, Input/Select Kelas Saat Ini
         - Kolom: NIS, Nama, JK, Kelas Saat Ini
         - Setiap baris clickable → klik → pindah dari availableStudents ke staging (hapus dari local availableStudents, tambah ke staging)
         - Data yang ditampilkan = availableStudents yang lolos semua filter DAN belum di staging
         - Row cursor-pointer dengan hover highlight

       - **Tabel kanan header:** "Akan Dipindahkan ({n})"
         - Kolom: NIS, Nama, JK, Aksi
         - Aksi per baris: Button "Batal" (size sm) → kembalikan ke availableStudents, hapus dari staging
         - Empty state: "Klik siswa di kiri untuk menambahkan"

    6. **Action bar bawah** (tampil jika staging.length > 0):
       - Button "Proses ({n} siswa)" → disabled saat loading
       - Proses logic:
         ```
         setLoading(true)
         await Promise.all(staging.map(s => {
           if (s.enrollmentId) {
             return apiClient.patch(`/master/student-classes/${s.enrollmentId}/transfer`, { classId: selectedClassId })
           } else {
             return apiClient.post('/master/student-classes', { studentId: s.id, classId: selectedClassId, schoolYearId: selectedYearId })
           }
         }))
         setStaging([])
         // refresh currentMembers
         // refresh availableStudents
         setLoading(false)
         ```

    **API calls:**
    - GET /master/school-years → SchoolYear[] (di-load sekali saat mount)
    - GET /master/school-units → SchoolUnit[] (di-load sekali saat mount)
    - GET /master/classes?school_unit_id=X → Class[] (saat unit berubah)
    - GET /master/student-classes?classId=X&schoolYearId=Y → ClassMember[] (saat kelas berubah)
    - GET /master/students/mapping?schoolYearId=Y&excludeClassId=X → MappingAvailableStudent[] (saat kelas berubah)
    - DELETE /master/student-classes/:id (aksi Lepas)
    - PATCH /master/student-classes/:id/transfer (proses transfer)
    - POST /master/student-classes (proses enroll baru)

    **Ikuti pola UI dari halaman kelas lain:**
    - Gunakan className table pattern yang sama dengan index.tsx di master/kelas
    - Format gender: 'L' → 'Laki-laki', 'P' → 'Perempuan' (inline, tanpa shared util)
    - Gunakan shadcn Select untuk dropdown, Input untuk filter

    AVOID: Jangan import dari halaman lain, tidak perlu shared util baru

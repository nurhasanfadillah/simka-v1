<objective>
## Goal
Bangun AcademicModule baru dengan 2 endpoint: GET /academic/promotion/preview (query siswa + flag tunggakan) dan POST /academic/promotion (proses massal naik/tinggal/lulus dalam satu DB transaction).

## Purpose
Operator perlu cara untuk menaikan kelas seluruh siswa ke tahun ajaran berikutnya secara massal — tanpa harus enroll satu per satu. Preview memungkinkan review sebelum eksekusi, proses massal menjamin konsistensi via transaction.

## Output
- Module baru `academic/` dengan sub-module `promotion/`
- GET /academic/promotion/preview — list siswa di kelas+tahun tertentu beserta flag hasTunggakan
- POST /academic/promotion — proses batch: naik (enroll baru), tinggal (enroll + set mengulang), lulus (update studentStatus)
- AcademicModule terdaftar di AppModule
</objective>

  <action>
    **dto/promote.dto.ts:**
    ```typescript
    class PromoteItemDto {
      studentId: number       // IsInt, IsPositive
      action: 'naik' | 'tinggal' | 'lulus'  // IsEnum
      toClassId?: number      // IsOptional, IsInt — required jika action = naik|tinggal
    }

    class PromoteDto {
      fromYearId: number      // IsInt, IsPositive
      toYearId: number        // IsInt, IsPositive
      items: PromoteItemDto[] // IsArray, ValidateNested, ArrayMinSize(1)
    }
    ```
    Validasi custom: jika action !== 'lulus' maka toClassId wajib ada.
    Lakukan validasi ini di service (bukan DTO), throw BadRequestException jika invalid.

    **promotion.service.ts — method promote(dto, operatorId):**
    Jalankan dalam db.transaction() Drizzle:

    Loop setiap item:
    - action === 'naik':
      INSERT INTO student_classes { studentId, classId: toClassId, schoolYearId: toYearId }
      Jika unique constraint violation (23505) → tambah ke errors array, lanjut (jangan throw)

    - action === 'tinggal':
      INSERT INTO student_classes { studentId, classId: toClassId, schoolYearId: toYearId }
      UPDATE students SET registrationStatus = 'mengulang' WHERE id = studentId
      Handle unique constraint violation sama seperti naik

    - action === 'lulus':
      UPDATE students SET studentStatus = 'lulus' WHERE id = studentId

    Return:
    ```typescript
    {
      processed: number,
      naik: number,
      tinggal: number,
      lulus: number,
      errors: Array<{ studentId, message }>
    }
    ```

    **promotion.controller.ts:**
    @Post() → promote(@Body() dto, @Req() req)
    Guard: @RequirePermission('student.manage')
    Inject req.user.id sebagai operatorId (untuk logging jika perlu di masa depan)

    Gunakan db (DRIZZLE inject token) langsung di service — ikuti pattern yang sama seperti service lain di codebase (inject via constructor dengan @Inject(DATABASE_CONNECTION) atau sejenisnya — cek service lain untuk pattern exact-nya).

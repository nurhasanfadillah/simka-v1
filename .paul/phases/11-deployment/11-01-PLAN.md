    1. Buat direktori `nginx/` di root proyek, lalu buat `nginx/simka.conf`:
    ```nginx
    server {
        listen 80;
        server_name 129.226.213.135;

        root /var/www/simka/apps/frontend/dist;
        index index.html;

        # Frontend SPA — semua route diarahkan ke index.html
        location / {
            try_files $uri $uri/ /index.html;
        }

        # Backend API proxy
        location /api {
            proxy_pass http://localhost:3000;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection 'upgrade';
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_cache_bypass $http_upgrade;

            # Puppeteer PDF bisa lambat — naikkan timeout
            proxy_read_timeout 120s;
            proxy_connect_timeout 10s;
        }
    }
    ```

    2. Buat `ecosystem.config.js` di root proyek:
    ```js
    module.exports = {
      apps: [
        {
          name: 'simka-backend',
          script: 'dist/main.js',
          cwd: '/var/www/simka/apps/backend',
          instances: 1,
          autorestart: true,
          watch: false,
          max_memory_restart: '512M',
          env_production: {
            NODE_ENV: 'production',
            PORT: 3000,
          },
        },
      ],
    }
    ```

    3. Buat `apps/backend/.env.production.example`:
    ```
    # Database
    DATABASE_URL=postgresql://simka:GANTI_PASSWORD_DB@localhost:5432/simka_db

    # Redis
    REDIS_HOST=localhost
    REDIS_PORT=6379

    # JWT — generate dengan: node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
    JWT_ACCESS_SECRET=GANTI_DENGAN_HEX_64_BYTES
    JWT_REFRESH_SECRET=GANTI_DENGAN_HEX_64_BYTES_BERBEDA
    JWT_ACCESS_EXPIRES_IN=15m
    JWT_REFRESH_EXPIRES_IN=7d

    # App
    PORT=3000
    NODE_ENV=production
    FRONTEND_URL=http://129.226.213.135
    ```

    Avoid: jangan buat .env.production (tanpa .example) — file secrets tidak boleh masuk ke repo.
  </action>

<task type="auto">
  <name>Task 2: Buat DEPLOY.md — runbook deployment VPS Ubuntu</name>
  <files>DEPLOY.md</files>
  <action>
    Buat `DEPLOY.md` di root proyek dengan konten berikut (runbook lengkap):

    ```markdown
    # DEPLOY.md — SIMKA Deployment Guide

    Target VPS: ubuntu@129.226.213.135
    Stack: NestJS (pm2) + React (Nginx static) + PostgreSQL + Redis

    ---

    ## 1. Koneksi ke VPS

    ```bash
    ssh ubuntu@129.226.213.135
    # masukkan password saat diminta
    ```

    ## 2. Install Dependencies (sekali saja)

    ```bash
    # Update sistem
    sudo apt update && sudo apt upgrade -y

    # Install Node.js 20 via NodeSource
    curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
    sudo apt install -y nodejs

    # Install pnpm
    npm install -g pnpm

    # Install pm2
    npm install -g pm2

    # Install Nginx
    sudo apt install -y nginx
    sudo systemctl enable nginx
    sudo systemctl start nginx

    # Install PostgreSQL
    sudo apt install -y postgresql postgresql-contrib
    sudo systemctl enable postgresql
    sudo systemctl start postgresql

    # Install Redis
    sudo apt install -y redis-server
    sudo systemctl enable redis-server
    sudo systemctl start redis-server

    # Install Puppeteer dependencies (untuk PDF)
    sudo apt install -y \
      chromium-browser \
      fonts-liberation \
      libasound2 \
      libatk-bridge2.0-0 \
      libatk1.0-0 \
      libcups2 \
      libdrm2 \
      libgbm1 \
      libgtk-3-0 \
      libnspr4 \
      libnss3 \
      libxcomposite1 \
      libxdamage1 \
      libxfixes3 \
      libxrandr2 \
      xdg-utils
    ```

    ## 3. Setup PostgreSQL

    ```bash
    sudo -u postgres psql

    -- Di dalam psql:
    CREATE USER simka WITH PASSWORD 'GANTI_PASSWORD_KUAT';
    CREATE DATABASE simka_db OWNER simka;
    GRANT ALL PRIVILEGES ON DATABASE simka_db TO simka;
    \q
    ```

    ## 4. Upload Kode ke VPS

    **Dari komputer lokal (bukan dari VPS):**

    ```bash
    # Buat direktori di VPS
    ssh ubuntu@129.226.213.135 "sudo mkdir -p /var/www/simka && sudo chown ubuntu:ubuntu /var/www/simka"

    # Upload kode (jalankan dari root proyek simka-v1/)
    rsync -avz --exclude=node_modules --exclude=.env --exclude='apps/backend/dist' --exclude='apps/frontend/dist' \
      . ubuntu@129.226.213.135:/var/www/simka/
    ```

    ## 5. Setup Environment Backend

    ```bash
    # Di VPS:
    cp /var/www/simka/apps/backend/.env.production.example /var/www/simka/apps/backend/.env

    # Edit dengan nilai production nyata:
    nano /var/www/simka/apps/backend/.env
    ```

    Isi nilai:
    - `DATABASE_URL` → pakai user/password dari step 3
    - `JWT_ACCESS_SECRET` → generate: `node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"`
    - `JWT_REFRESH_SECRET` → generate ulang dengan perintah sama
    - `FRONTEND_URL=http://129.226.213.135`

    ## 6. Install Dependencies & Build

    ```bash
    cd /var/www/simka

    # Install semua dependencies
    pnpm install --frozen-lockfile

    # Build backend + frontend (via turbo)
    pnpm build
    ```

    Hasil build:
    - Backend: `apps/backend/dist/`
    - Frontend: `apps/frontend/dist/`

    ## 7. Jalankan Database Migration + Seed

    ```bash
    cd /var/www/simka/apps/backend

    # Jalankan migration
    pnpm db:migrate

    # Seed data awal (roles, permissions, Super Admin)
    pnpm db:seed
    ```

    Setelah seed, cek credentials Super Admin di output seed atau di file seed.

    ## 8. Start Backend dengan pm2

    ```bash
    cd /var/www/simka

    # Start backend
    pm2 start ecosystem.config.js --env production

    # Simpan pm2 config agar auto-start setelah reboot
    pm2 save
    pm2 startup
    # Jalankan perintah yang muncul dari output pm2 startup

    # Cek status
    pm2 list
    pm2 logs simka-backend --lines 20
    ```

    Backend harus online. Test:
    ```bash
    curl http://localhost:3000/api
    # Harus return response (bukan connection refused)
    ```

    ## 9. Setup Nginx

    ```bash
    # Copy config
    sudo cp /var/www/simka/nginx/simka.conf /etc/nginx/sites-available/simka

    # Enable site
    sudo ln -s /etc/nginx/sites-available/simka /etc/nginx/sites-enabled/simka

    # Hapus default site (opsional)
    sudo rm -f /etc/nginx/sites-enabled/default

    # Test config
    sudo nginx -t

    # Reload Nginx
    sudo systemctl reload nginx
    ```

    ## 10. Buka Firewall

    ```bash
    sudo ufw allow OpenSSH
    sudo ufw allow 'Nginx Full'
    sudo ufw enable
    sudo ufw status
    ```

    ## 11. Smoke Test

    Buka browser → http://129.226.213.135

    Alur test:
    1. Login dengan kredensial Super Admin dari seed
    2. Cek Dashboard — stat cards muncul
    3. Buka Master → Tahun Pelajaran
    4. Buka Keuangan → Data Tagihan
    5. Buka Laporan → Laporan Harian
    6. Buka Pengaturan → Profil (lihat data user)

    ---

    ## Update Deployment (setelah ada perubahan kode)

    ```bash
    # Di komputer lokal:
    rsync -avz --exclude=node_modules --exclude=.env --exclude='apps/backend/dist' --exclude='apps/frontend/dist' \
      . ubuntu@129.226.213.135:/var/www/simka/

    # Di VPS:
    cd /var/www/simka
    pnpm install --frozen-lockfile
    pnpm build
    pm2 restart simka-backend
    ```

    ---

    ## Troubleshooting

    | Problem | Cek |
    |---------|-----|
    | 502 Bad Gateway | `pm2 logs simka-backend` — backend crashed? |
    | Frontend blank | `pm2 logs` + browser console errors |
    | DB connection error | `.env` DATABASE_URL benar? PostgreSQL running? `sudo systemctl status postgresql` |
    | Redis error | `sudo systemctl status redis-server` |
    | PDF gagal generate | Puppeteer deps terpasang? `chromium-browser --version` |
    ```

    Avoid: jangan tulis password VPS di DEPLOY.md.

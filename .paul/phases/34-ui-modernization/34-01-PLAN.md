<objective>
## Goal
Membangun design system foundation yang menjadi prasyarat semua UI polish — single source of truth untuk warna, tipografi, spacing, radius, shadow, dan animasi.

## Purpose
Saat ini SIMKA punya dua source warna (hardcoded hex di tailwind.config.ts + CSS vars di :root) dan tidak ada standar tipografi/spacing yang konsisten. Semua halaman akan dipoles menggunakan token dari foundation ini.

## Output
1. CSS variables terkonsolidasi (warna + typography + spacing + radius + shadow) di `index.css`
2. Tailwind config sinkron dengan CSS vars
3. Font `Inter` atau `Plus Jakarta Sans` terintegrasi
4. Plugin `tailwindcss-animate` terpasang (fix bug: class animate-in/fade-in sudah digunakan tapi plugin missing)
5. CSS custom animation keyframes (fade, slide, scale)
6. UI store Zustand untuk state sidebar collapsed + theme
7. Custom utility classes di `@layer components` (glass cards, page transitions)
8. Border-radius token system (sharp default, subtle round untuk card/input)
</objective>

<acceptance_criteria>
## AC-1: CSS Variables Consolidated
Given color tokens digunakan di seluruh app
When developer membuka `index.css`
Then SEMUA warna didefinisikan sebagai CSS vars di `:root` — termasuk primary, accent, gray scale, semantic colors — dan tailwind.config.ts mereferensi vars tersebut (bukan hardcoded hex)

## AC-2: Typography Scale Tersedia
Given halaman menggunakan heading, body, label, caption
When text dirender
Then ukuran font mengikuti scale token yang konsisten (`text-xs` s/d `text-3xl` terdefinisi di `index.css` `@layer base`)

## AC-3: Spacing System Tersedia
Given komponen menggunakan padding dan margin
When layout dirender
Then spacing mengikuti token `--spacing-xs` s/d `--spacing-xl` di CSS vars

## AC-4: Border Radius System
Given mayoritas UI sharp, hanya card dan input yang round
When komponen dirender
Then `--radius` default = `0.125rem` (extra small), `--radius-card` = `0.5rem`, `--radius-input` = `0.375rem`

## AC-5: Shadow/Elevation System
Given card, dialog, dropdown
When komponen dirender
Then shadow mengikuti level 1-3: `--shadow-sm` (card), `--shadow-md` (dialog), `--shadow-lg` (dropdown)

## AC-6: Font Inter Terpasang
Given app dimuat di browser
When developer inspect computed styles
Then font-family = `'Inter', system-ui, sans-serif`

## AC-7: tailwindcss-animate Plugin Active
Given animasi `animate-in`, `fade-in`, `slide-in-from-left` digunakan di dialog/dropdown
When komponen dibuka
Then animasi berjalan dengan benar (tidak instant appear)

## AC-8: UI Store Tersedia
Given developer perlu toggle sidebar
When `useUIStore` dipanggil
Then tersedia state `sidebarCollapsed: boolean` + action `toggleSidebar()` + persist ke localStorage

## AC-9: Build Pass

Then TypeScript compile + Vite build sukses tanpa error

<task type="auto">
  <name>Create UI Zustand Store</name>
  <files>apps/frontend/src/stores/ui.store.ts</files>
  <action>
    Buat file baru `stores/ui.store.ts`:
    ```
    import { create } from 'zustand'
    import { persist } from 'zustand/middleware'

    interface UIState {
      sidebarCollapsed: boolean
      toggleSidebar: () => void
      setSidebarCollapsed: (collapsed: boolean) => void
    }

    export const useUIStore = create<UIState>()(
      persist(
        (set) => ({
          sidebarCollapsed: false,
          toggleSidebar: () => set((s) => ({ sidebarCollapsed: !s.sidebarCollapsed })),
          setSidebarCollapsed: (collapsed) => set({ sidebarCollapsed: collapsed }),
        }),
        { name: 'simka-ui' }
      )
    )
    ```
    Gunakan pattern yang sama dengan `auth.store.ts` (zustand v5 + persist middleware).

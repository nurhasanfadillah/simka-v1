<objective>
## Goal
Menambahkan filter "Tanpa Kelas" di halaman Mapping Kelas — memungkinkan operator melihat dan meng-assign siswa yang belum masuk kelas manapun.

## Purpose
Saat ini, siswa tanpa kelas tidak muncul di tabel "Siswa Tersedia" karena bug di WHERE clause backend (`ne(studentClasses.classId, excludeClassId)` mengeksklusi NULL). Fix ini memastikan siswa tanpa kelas tampil dan bisa difilter dengan opsi "Tanpa Kelas".

## Output
- Backend: `findAvailableForMapping` mengembalikan siswa tanpa kelas
- Frontend: dropdown "Kelas Saat Ini" punya opsi "Tanpa Kelas"
</objective>

<task type="auto">
  <name>Fix backend WHERE clause — include unassigned students</name>
  <files>apps/backend/src/master/students/students.service.ts</files>
  <action>
    Di method `findAvailableForMapping`, ubah WHERE clause dari:
    ```
    and(
      eq(students.studentStatus, 'aktif'),
      ne(studentClasses.classId, excludeClassId),
    )
    ```
    menjadi:
    ```
    and(
      eq(students.studentStatus, 'aktif'),
      or(
        isNull(studentClasses.classId),
        ne(studentClasses.classId, excludeClassId),
      ),
    )
    ```
    Alasan: `ne(NULL, value)` di PostgreSQL menghasilkan NULL (bukan boolean), sehingga row di-skip.
    `or(isNull(...), ne(...))` memastikan siswa tanpa kelas tetap muncul.

    Import `or` dan `isNull` sudah tersedia dari `drizzle-orm` (cek line 10).
  </action>
  <verify>Restart backend, buka halaman Mapping Kelas — siswa tanpa kelas muncul di tabel "Siswa Tersedia" dengan kolom "Kelas Saat Ini" = "—"</verify>
  <done>AC-1 satisfied: siswa tanpa kelas muncul di hasil mapping</done>
</task>

Tambahkan modal konfirmasi sebelum aksi Proses di halaman Mapping Kelas — user harus mengkonfirmasi jumlah siswa yang akan di-assign sebelum request dikirim ke server.

<acceptance_criteria>
## AC-1: Tombol Proses membuka modal konfirmasi
Given user sudah memilih kelas target dan menambahkan siswa ke staging
When user mengklik tombol "Proses (N siswa)"
Then modal konfirmasi muncul dengan jumlah siswa dan nama kelas tujuan

## AC-2: Modal menampilkan info yang relevan
Given modal konfirmasi terbuka
When user melihat isi modal
Then tampil: jumlah siswa yang akan di-assign, nama kelas tujuan, dan dua tombol (Batal + Konfirmasi)

## AC-3: Konfirmasi menjalankan proses, Batal menutup modal
Given modal konfirmasi terbuka
When user klik "Konfirmasi"
Then handleProses dijalankan dan modal ditutup
When user klik "Batal" atau menutup modal
Then modal ditutup, staging tetap tidak berubah
</acceptance_criteria>

<tasks>
<task type="auto">
  <name>Task 1: Tambah modal konfirmasi Proses di mapping.tsx</name>
  <files>apps/frontend/src/pages/master/kelas/mapping.tsx</files>
  <action>
    1. Import komponen Dialog dari shadcn/ui:
       ```
       import {
         Dialog,
         DialogContent,
         DialogHeader,
         DialogTitle,
         DialogFooter,
         DialogDescription,
       } from '@/components/ui/dialog'
       ```

    2. Tambah state `confirmOpen`:
       ```
       const [confirmOpen, setConfirmOpen] = useState(false)
       ```

    3. Ubah tombol "Proses" di action bar agar membuka modal (bukan langsung memanggil handleProses):
       ```
       onClick={() => setConfirmOpen(true)}
       ```
       Hapus `disabled={loading}` dari tombol ini (loading state ada di dalam modal).

    4. Update handleProses agar menutup modal setelah selesai:
       Tambah `setConfirmOpen(false)` di awal handleProses (sebelum setLoading(true)),
       atau di finally block setelah setLoading(false).
       Letakkan di finally agar modal tutup baik sukses maupun gagal.

    5. Tambahkan komponen Dialog di JSX (sebelum closing `</div>` terakhir):
       ```jsx
       <Dialog open={confirmOpen} onOpenChange={setConfirmOpen}>
         <DialogContent>
           <DialogHeader>
             <DialogTitle>Konfirmasi Proses</DialogTitle>
             <DialogDescription>
               {staging.length} siswa akan ditambahkan ke kelas {selectedClassName}.
               Tindakan ini tidak dapat dibatalkan secara massal.
             </DialogDescription>
           </DialogHeader>
           <DialogFooter>
             <Button variant="outline" onClick={() => setConfirmOpen(false)} disabled={loading}>
               Batal
             </Button>
             <Button
               onClick={handleProses}
               disabled={loading}
               className="bg-primary hover:bg-primary/90"
             >
               {loading ? 'Memproses...' : 'Konfirmasi'}
             </Button>
           </DialogFooter>
         </DialogContent>
       </Dialog>
       ```

    Jangan ubah logika handleProses, staging, atau bagian lain halaman.
  </action>
  <verify>
    - Klik tombol Proses → modal muncul dengan info jumlah siswa dan kelas
    - Klik Batal → modal tutup, staging tidak berubah
    - Klik Konfirmasi → request dikirim, modal tutup setelah selesai
  </verify>
  <done>AC-1, AC-2, AC-3 satisfied: modal konfirmasi berfungsi sebelum aksi Proses</done>
</task>

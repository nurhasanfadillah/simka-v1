<objective>
## Goal
Mengganti pencarian siswa inline di halaman Transaksi Baru menjadi Dialog pencarian siswa dengan filter tahun pelajaran, unit sekolah, dan kelas. Hasil pencarian ditampilkan dalam tabel (NIS, Nama, Kelas, Unit) — klik baris siswa membuka dialog transaksi (PLAN 33-02).

## Purpose
Saat ini pencarian siswa memuat seluruh data lalu filter client-side, tanpa filter tahun ajaran/unit/kelas. Dialog dengan filter server-side memungkinkan operator mencari siswa dengan lebih presisi, terutama di sekolah multi-unit.

## Output
- Backend: endpoint `GET /master/students/search?q=&schoolYearId=&unitId=&classId=` — search siswa aktif dengan filter opsional
- Frontend: Dialog komponen pencarian siswa dengan 3 dropdown filter + 1 input teks + tabel hasil
- Halaman Transaksi Baru membuka dialog ini saat tombol "Transaksi Baru" diklik
- Klik baris siswa di dialog → menutup dialog dan membuka dialog transaksi (dihandle di PLAN 33-02)
</objective>

## AC-1: Endpoint search siswa dengan filter
```gherkin
Given parameter `q` (nama/NIS), `schoolYearId` (opsional), `unitId` (opsional), `classId` (opsional)
When GET /master/students/search dipanggil
Then mengembalikan array siswa aktif dengan kolom: id, nis, name, activeClassName, activeUnitName
  Dan hasil terbatas maksimal 50 baris
  Dan jika schoolYearId diberikan, hanya siswa yang memiliki enrollment di tahun tersebut
  Dan jika unitId diberikan, hanya siswa di unit tersebut
  Dan jika classId diberikan, hanya siswa di kelas tersebut
```

## AC-2: Dialog pencarian siswa terbuka dari halaman Transaksi Baru
```gherkin
Given user berada di halaman /keuangan/transaksi/baru
When halaman dimuat (atau setelah transaksi selesai)
Then tampil dialog "Pilih Siswa" dengan:
  - Input teks "Cari nama atau NIS"
  - Dropdown Tahun Pelajaran (default: tahun aktif)
  - Dropdown Unit Sekolah (default: semua)
  - Dropdown Kelas (default: semua, difilter berdasarkan unit terpilih jika ada)
  - Tabel hasil: NIS, Nama, Kelas, Unit
  - Tombol "Batal" untuk menutup dialog
```

## AC-3: Klik baris siswa menutup dialog
```gherkin
Given dialog pencarian siswa terbuka dengan hasil
When user mengklik baris siswa
Then dialog pencarian tertutup
  Dan state `selectedStudent` terisi dengan data siswa yang dipilih
  Dan (di PLAN 33-02) dialog transaksi akan terbuka
```

## AC-4: Filter berjenjang (unit mempengaruhi kelas)
```gherkin
Given dialog pencarian terbuka
When user memilih Unit Sekolah tertentu
Then dropdown Kelas hanya menampilkan kelas di unit tersebut
```

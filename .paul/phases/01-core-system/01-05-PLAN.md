<objective>
## Goal
CRUD API untuk payment_posts — resource master data terakhir di Phase 01 — terproteksi JWT + RBAC, mengikuti pola yang sudah established.

## Purpose
Payment posts (POS pembayaran) adalah template kategori tagihan: SPP, Uang Gedung, Seragam, Buku, dll. Tanpa POS, billing module tidak bisa membuat payment_templates maupun generate tagihan. Ini adalah prerequisite final sebelum Phase 02 (Billing Engine).

## Output
- `GET/POST/PATCH /api/master/payment-posts` dan `/api/master/payment-posts/:id`
- type enum: `bulanan` (generate per bulan) atau `bebas` (nominal tetap)
- Semua endpoint terproteksi JWT + RBAC
</objective>

<acceptance_criteria>

## AC-1: PaymentPosts CRUD berfungsi
```gherkin
Given user dengan permission payment_post.view
When GET /api/master/payment-posts
Then response 200 dengan array semua POS (termasuk 4 dari seed)

Given user dengan permission payment_post.view
When GET /api/master/payment-posts/:id
Then response 200 dengan data POS, 404 jika tidak ada

Given user dengan permission payment_post.create
When POST /api/master/payment-posts dengan { code, name, type } valid
Then response 201 dengan data POS yang dibuat

Given user dengan permission payment_post.update
When PATCH /api/master/payment-posts/:id dengan body partial
Then response 200 dengan data yang diupdate

Given code sudah dipakai POS lain
When POST atau PATCH dengan code yang sama
Then response 409 Conflict "Kode POS sudah dipakai"
```

## AC-2: RBAC memproteksi semua endpoint
```gherkin
Given request tanpa Authorization header
When request ke endpoint manapun di /api/master/payment-posts*
Then response 401 Unauthorized

Given user terautentikasi tanpa permission payment_post.view
When GET /api/master/payment-posts
Then response 403 Forbidden
```

      ```typescript
        @IsString() @IsNotEmpty() @MaxLength(20)
        @Transform(({ value }) => value.toUpperCase())

        @IsString() @IsNotEmpty() @MaxLength(255)
        name: string

        @IsEnum(['bulanan', 'bebas'])
        type: 'bulanan' | 'bebas'

        @IsString() @IsOptional()

      }
      ```

    **3. PaymentPostsService** (`payment-posts.service.ts`):
    - Inject `DRIZZLE`
    - Helper: `function isUniqueViolation(err): boolean` — cek `err?.code ?? err?.cause?.code === '23505'`
    - `findAll()`: SELECT * FROM payment_posts ORDER BY name ASC

    - `create(dto)`: INSERT, tangkap 23505 → ConflictException('Kode POS sudah dipakai')
    - `update(id, dto)`: findOne dulu → UPDATE, tangkap 23505 → ConflictException('Kode POS sudah dipakai')

    - `POST /` → `@RequirePermissions('payment_post.create')` → create(dto), return 201
    - `PATCH /:id` → `@RequirePermissions('payment_post.update')` → update(id, dto)

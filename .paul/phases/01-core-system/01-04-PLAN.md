<objective>
## Goal
CRUD API untuk students (dengan NIS auto-generate) dan enrollment API untuk student_classes — semua terproteksi JWT + RBAC, mengikuti pola yang sudah established di plan 01-03.

## Purpose
Siswa adalah entitas inti sistem keuangan — tanpa data siswa, tidak bisa generate tagihan maupun transaksi. Plan ini menyelesaikan dua resource terakhir di layer master data akademik sebelum beralih ke payment_posts.

## Output
- `GET/POST/PATCH /api/master/students` dan `/api/master/students/:id`
- `GET /api/master/students/:id/classes` — riwayat kelas siswa
- `GET/POST /api/master/student-classes` — enrollment dan list
- `PATCH /api/master/student-classes/:id/transfer` — pindah kelas dalam tahun ajaran yang sama
- NIS di-generate otomatis: `3randomUppercaseChars + ddmmyy(birthDate) + 2digitEntryYear`
</objective>

    **3. NIS Generation** (helper dalam service):
    ```typescript
    // Format: 3 random uppercase + ddmmyy(birthDate) + 2-digit entryYear
    // Contoh: ABC300526 26 (lahir 30/05/XXXX, masuk 2026) → "ABC300526 26"
    private generateNis(birthDate: string, entryYear: number): string {
      const alpha = 'ABCDEFGHIJKLMNOPQRSTUVWXYZ';
      const rand3 = Array.from({length: 3}, () => alpha[Math.floor(Math.random() * 26)]).join('');
      const d = new Date(birthDate);
      const dd = String(d.getDate()).padStart(2, '0');
      const mm = String(d.getMonth() + 1).padStart(2, '0');
      const yy = String(d.getFullYear()).slice(-2);
      const ey = String(entryYear).slice(-2);
      return `${rand3}${dd}${mm}${yy}${ey}`;
    }
    ```
    NIS uniqueness: jika terjadi collision (sangat jarang), retry 1x sebelum throw InternalServerErrorException.

    **4. StudentsService** (`students.service.ts`):
    - Inject `DRIZZLE`
    - `findAll(schoolUnitId?: number)`:
      - SELECT students.*, student_classes.class_id, classes.name as class_name, school_units.name as unit_name
        FROM students
        LEFT JOIN student_classes ON student_classes.student_id = students.id AND student_classes.is_active = true
        LEFT JOIN classes ON classes.id = student_classes.class_id
        LEFT JOIN school_units ON school_units.id = classes.school_unit_id
      - Jika `schoolUnitId` diberikan: WHERE school_units.id = schoolUnitId
      - ORDER BY students.name ASC
    - `findOne(id)`: SELECT dengan LEFT JOIN ke student_classes aktif, throw NotFoundException jika tidak ada
    - `create(dto)`:
      - Generate NIS dengan `generateNis(dto.birthDate, dto.entryYear)`
      - INSERT ke students, tangkap error 23505:
        - Jika kolom nisn: ConflictException('NISN sudah dipakai')
        - Lainnya: ConflictException('NIS sudah dipakai') — NIS collision, retry 1x
    - `update(id, dto)`:
      - findOne dulu
      - UPDATE field yang diberikan (exclude nis dan entryYear dari PartialType)
      - Tangkap 23505 pada nisn → ConflictException('NISN sudah dipakai')

<objective>
## Goal
Implementasi Auth Backend lengkap: login dengan JWT, refresh token disimpan di Redis, JWT Guard global, dan RBAC Guard berbasis permission code.

## Purpose
Fondasi keamanan sistem — semua endpoint akan diproteksi oleh JWT Guard, dan akses per fitur dikontrol oleh RBAC Guard menggunakan permission codes yang sudah di-seed.

## Output
- AuthModule fungsional (login, refresh, logout, me)
- JWT Guard aktif global (semua route private kecuali yang di-exclude)
- RBAC Guard siap dipakai via `@RequirePermissions('code')` decorator
</objective>

<acceptance_criteria>

## AC-1: Login berhasil
```gherkin
Given user admin@alhasaniyyah.sch.id dengan password Admin123! ada di DB
When POST /auth/login dengan { email, password } valid
Then response 200 dengan { access_token, refresh_token, user: { id, name, email, role } }
```

## AC-2: Login gagal ditolak
```gherkin
Given kredensial salah atau user tidak aktif
When POST /auth/login
Then response 401 dengan message yang informatif
```

## AC-3: Refresh token menghasilkan access token baru
```gherkin
Given refresh_token valid yang diperoleh dari login
When POST /auth/refresh dengan { refresh_token }
Then response 200 dengan { access_token } baru (15 menit)
```

## AC-4: Endpoint private membutuhkan JWT valid
```gherkin
Given endpoint GET /auth/me (protected)
When request tanpa Authorization header atau token expired
Then response 401
When request dengan Bearer token valid
Then response 200 dengan data current user + permissions
```

## AC-5: RBAC Guard memblokir akses tanpa permission
```gherkin
Given endpoint yang diberi @RequirePermissions('user.create')
When user tidak memiliki permission 'user.create'
Then response 403
When user memiliki permission tersebut
Then request diteruskan ke handler
```

<task type="auto">
  <name>Task 1: Install packages + Redis provider + AuthService</name>
  <files>
    apps/backend/package.json,
    apps/backend/src/redis/redis.module.ts,
    apps/backend/src/redis/redis.provider.ts,
    apps/backend/src/auth/auth.module.ts,
    apps/backend/src/auth/auth.service.ts,
    apps/backend/src/auth/dto/login.dto.ts
  </files>
  <action>
    **1. Install packages** (jalankan dari apps/backend/):
    ```
    pnpm add @nestjs/jwt @nestjs/passport passport passport-jwt ioredis
    pnpm add -D @types/passport-jwt @types/passport
    ```

    **2. Buat RedisModule** (`src/redis/`):
    - `redis.provider.ts`: provider dengan token `REDIS_CLIENT`, buat instance `new Redis({ host, port })` dari env `REDIS_HOST` (default: localhost) dan `REDIS_PORT` (default: 6379)
    - `redis.module.ts`: `@Global()` module, export `REDIS_CLIENT` provider

    **3. Buat login DTO** (`src/auth/dto/login.dto.ts`):
    - Class `LoginDto` dengan field `email: string` dan `password: string`
    - Gunakan class-validator: `@IsEmail()`, `@IsString()`, `@IsNotEmpty()`
    - Install `class-validator` dan `class-transformer` juga jika belum ada
    - Aktifkan `ValidationPipe` global di `main.ts`

    **4. Buat AuthService** (`src/auth/auth.service.ts`):
    - Inject `DRIZZLE` (database) dan `REDIS_CLIENT` (ioredis) dan `JwtService`
    - Method `validateUser(email, password)`: query user + role dari DB, bandingkan password dengan bcrypt.compare, return user object atau null
    - Method `login(user)`:
      - Generate `access_token` via `jwtService.sign({ sub: user.id, email: user.email, roleId: user.roleId }, { expiresIn: '15m' })`
      - Generate `refresh_token` sebagai `randomUUID()`
      - Simpan di Redis: key `refresh:${user.id}`, value refresh_token, TTL 7 hari (604800 detik)
      - Return `{ access_token, refresh_token, user: { id, name, email, role: user.role.name } }`
    - Method `refreshToken(refreshToken)`:
      - Decode JWT untuk dapat userId (tanpa verify karena mungkin expired — gunakan `jwtService.decode()`)
      - Cek Redis: `GET refresh:${userId}` === refreshToken
      - Jika cocok, generate `access_token` baru
      - Return `{ access_token }`
      - Jika tidak cocok: throw `UnauthorizedException`
    - Method `logout(userId)`: `DEL refresh:${userId}` dari Redis
    - Method `getProfile(userId)`: query user + role + permissions dari DB

    **5. Buat AuthModule skeleton** (`src/auth/auth.module.ts`):
    - Import `JwtModule.register({ secret: process.env.JWT_SECRET, signOptions: { expiresIn: '15m' } })`
    - Import `PassportModule`
    - Providers: `AuthService` (strategy dan guards ditambah di task berikutnya)
    - Controllers: `AuthController` (dibuat di task berikutnya)

    **6. Tambah env vars ke `.env.example`:**
    ```
    JWT_SECRET=your-secret-key-here
    REDIS_HOST=localhost
    REDIS_PORT=6379
    ```

    Avoid:
    - Jangan simpan refresh token sebagai JWT — gunakan UUID random agar bisa di-revoke
    - Jangan gunakan synchronize JWT — selalu async/await
    - Jangan hardcode JWT_SECRET
  </action>
  <verify>
    `pnpm build` di apps/backend tidak error — AuthModule, RedisModule, AuthService ter-compile
  </verify>
  <done>AuthService dengan login/refresh/logout/getProfile ter-compile, Redis provider terdaftar global</done>
</task>

<task type="auto">
  <name>Task 2: JWT Strategy + JWT Guard + RBAC Guard + Decorators</name>
  <files>
    apps/backend/src/auth/strategies/jwt.strategy.ts,
    apps/backend/src/auth/guards/jwt.guard.ts,
    apps/backend/src/auth/guards/rbac.guard.ts,
    apps/backend/src/auth/decorators/require-permissions.decorator.ts,
    apps/backend/src/auth/decorators/current-user.decorator.ts
  </files>
  <action>
    **1. JWT Strategy** (`src/auth/strategies/jwt.strategy.ts`):
    - Extend `PassportStrategy(Strategy, 'jwt')` dari `passport-jwt`
    - Constructor options:
      ```
      jwtFromRequest: ExtractJwt.fromAuthHeaderAsBearerToken()
      ignoreExpiration: false
      secretOrKey: process.env.JWT_SECRET
      ```
    - Method `validate(payload)`:
      - Query user dari DB by `payload.sub` dengan join ke role dan permissions (via role_permissions → permissions)
      - Return object `{ id, name, email, roleId, role: string, permissions: string[] }` — ini yang jadi `req.user`
      - Throw `UnauthorizedException` jika user tidak ditemukan atau `is_active = false`

    **2. JWT Guard** (`src/auth/guards/jwt.guard.ts`):
    - Extend `AuthGuard('jwt')`
    - Override `canActivate`: cek metadata `IS_PUBLIC_KEY` — jika true, return true (skip auth)
    - Decorator `@Public()` untuk mark endpoint publik: `SetMetadata('IS_PUBLIC', true)`
    - Simpan `@Public()` decorator di `src/auth/decorators/public.decorator.ts`

    **3. RBAC Guard** (`src/auth/guards/rbac.guard.ts`):
    - Implement `CanActivate`
    - Inject `Reflector`
    - Baca metadata `PERMISSIONS_KEY` dari handler dan class
    - Jika tidak ada metadata permission: return true (hanya butuh login)
    - Cek `req.user.permissions` contains semua required permissions
    - Jika tidak: throw `ForbiddenException`

    **4. Decorators:**
    - `require-permissions.decorator.ts`: `@RequirePermissions(...permissions: string[])` → `SetMetadata('PERMISSIONS', permissions)`
    - `current-user.decorator.ts`: `@CurrentUser()` → `createParamDecorator((data, ctx) => ctx.switchToHttp().getRequest().user)`

    **5. Register ke AuthModule providers:**
    Tambahkan `JwtStrategy`, `JwtAuthGuard`, `RbacGuard` ke providers AuthModule. Export semua guards.

    Avoid:
    - Jangan query permissions satu per satu — join sekaligus saat validate strategy
    - Jangan lupa export guards dari AuthModule agar bisa dipakai di module lain
  </action>

    `pnpm build` success. Semua class ter-compile tanpa error TypeScript.
  </verify>
  <done>JWT Strategy, JwtAuthGuard, RbacGuard, dan decorators ter-compile dan terdaftar di AuthModule</done>
</task>

<task type="auto">
  <name>Task 3: AuthController + Wiring ke AppModule</name>
  <files>
    apps/backend/src/auth/auth.controller.ts,
    apps/backend/src/app.module.ts
  </files>
  <action>
    **1. AuthController** (`src/auth/auth.controller.ts`):
    - `@Controller('auth')`
    - `POST /auth/login` — `@Public()`, body `LoginDto`:
      - Panggil `authService.validateUser(email, password)`
      - Jika null: throw `UnauthorizedException('Email atau password salah')`
      - Return `authService.login(user)`
    - `POST /auth/refresh` — `@Public()`, body `{ refresh_token: string }`:
      - Decode refresh_token untuk dapat userId (gunakan `jwtService.decode()`)
      - Panggil `authService.refreshToken(userId, refresh_token)`
      - Return `{ access_token }`
    - `POST /auth/logout` — protected (butuh JWT):
      - Ambil `@CurrentUser()` untuk dapat userId
      - Panggil `authService.logout(userId)`
      - Return `{ message: 'Logout berhasil' }`
    - `GET /auth/me` — protected:
      - Ambil `@CurrentUser()` 
      - Return `authService.getProfile(user.id)`

    **2. Update AppModule** (`src/app.module.ts`):
    - Import `AuthModule`, `RedisModule`
    - Register global guards via APP_GUARD:
      ```typescript
      { provide: APP_GUARD, useClass: JwtAuthGuard },
      { provide: APP_GUARD, useClass: RbacGuard },
      ```
    - Guards dijalankan berurutan: JwtAuthGuard dulu (cek token), lalu RbacGuard (cek permission)

    **3. Update `src/main.ts`:**
    - Aktifkan `ValidationPipe` global:
      ```typescript
      app.useGlobalPipes(new ValidationPipe({ whitelist: true, transform: true }))
      ```
    - Set global prefix: `app.setGlobalPrefix('api')`
    - Pastikan CORS enabled untuk development

    Avoid:
    - Jangan apply `@UseGuards` manual di controller — guards sudah global via APP_GUARD
    - Jangan lupa `@Public()` di login dan refresh, atau endpoint tersebut akan return 401
  </action>

    3. `curl http://localhost:3000/api/auth/me` tanpa token → 401
    4. `curl http://localhost:3000/api/auth/me -H "Authorization: Bearer {token}"` → 200 dengan data user + permissions

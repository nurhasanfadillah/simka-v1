<objective>
## Goal
CRUD API untuk tiga entitas master data pertama: school_units, school_years, classes — semua terproteksi JWT + RBAC, menggunakan pola yang konsisten dan siap dikonsumsi frontend.

## Purpose
Master data adalah fondasi operasional sistem — tanpa school_units, school_years, dan classes yang bisa dikelola, fitur billing dan transaksi tidak bisa berjalan. Plan ini menyelesaikan layer API master data pertama dari tiga kelompok.

## Output
- `GET/POST/PATCH /api/master/school-units` dan `/api/master/school-units/:id`
- `GET/POST/PATCH /api/master/school-years` dan `/api/master/school-years/:id` + `PATCH /:id/activate`
- `GET/POST/PATCH /api/master/classes` dan `/api/master/classes/:id`
- Semua endpoint terproteksi JWT + RBAC menggunakan permission codes yang sudah di-seed
</objective>

<acceptance_criteria>

## AC-1: School Units CRUD berfungsi
```gherkin
Given user terautentikasi dengan permission school_unit.view
When GET /api/master/school-units
Then response 200 dengan array semua school units

Given user dengan permission school_unit.create
When POST /api/master/school-units dengan { name, code } valid
Then response 201 dengan data school unit yang dibuat

Given user dengan permission school_unit.update
When PATCH /api/master/school-units/:id dengan body partial
Then response 200 dengan data yang sudah diupdate

Given code sudah dipakai unit lain
When POST atau PATCH dengan code yang sama
Then response 409 Conflict
```

## AC-2: School Years CRUD + aktivasi berfungsi
```gherkin
Given user dengan permission school_year.view
When GET /api/master/school-years
Then response 200 dengan array semua tahun ajaran, isActive terlihat

Given user dengan permission school_year.create
When POST /api/master/school-years dengan { name, startYear, endYear }
Then response 201, isActive default false

Given user dengan permission school_year.update
When PATCH /api/master/school-years/:id/activate
Then response 200, tahun ajaran tsb isActive=true, tahun ajaran lain isActive=false
```

## AC-3: Classes CRUD berfungsi
```gherkin
Given user dengan permission class.view
When GET /api/master/classes (optional ?school_unit_id=N)
Then response 200 dengan array kelas, termasuk nama school unit

Given user dengan permission class.create
When POST /api/master/classes dengan { schoolUnitId, name, level }
Then response 201 dengan data kelas yang dibuat

Given schoolUnitId tidak ada di DB
When POST /api/master/classes dengan schoolUnitId invalid
Then response 404 Not Found
```

## AC-4: RBAC memproteksi semua endpoint
```gherkin
Given user terautentikasi tanpa permission yang dibutuhkan
When request ke endpoint manapun di /api/master/*
Then response 403 Forbidden

Given request tanpa Authorization header
When request ke endpoint manapun di /api/master/*
Then response 401 Unauthorized
```

    - `POST /` → `@RequirePermissions('school_unit.create')` → create(dto), return 201
    - `PATCH /:id` → `@RequirePermissions('school_unit.update')` → update(id, dto)

      ```typescript
      class CreateSchoolYearDto {
        @IsString() @IsNotEmpty() @MaxLength(20)
        name: string  // contoh: "2025/2026"
        @IsInt() @Min(2000) @Max(2099)
        startYear: number
        @IsInt() @Min(2000) @Max(2099)
        endYear: number
      }
      ```
    - Validasi tambahan di service: endYear harus > startYear
    - `update-school-year.dto.ts`: PartialType(CreateSchoolYearDto)

    **2. SchoolYearsService** (`src/master/school-years/school-years.service.ts`):
    - `findAll()`: SELECT * FROM school_years ORDER BY start_year DESC
    - `findOne(id)`: SELECT by id, throw NotFoundException jika tidak ada
    - `create(dto)`:
      - Validasi endYear > startYear → BadRequestException
      - INSERT dengan isActive: false (default)
      - Tangkap unique constraint name → ConflictException('Nama tahun ajaran sudah ada')
    - `update(id, dto)`:
      - findOne dulu
      - UPDATE field yang diberikan
      - Tangkap unique constraint
    - `activate(id)`:
      - findOne dulu
      - Gunakan transaksi DB:
        1. UPDATE school_years SET is_active = false WHERE is_active = true (deaktifkan semua)
        2. UPDATE school_years SET is_active = true WHERE id = :id
      - Return tahun ajaran yang diaktifkan

    **3. SchoolYearsController**:
    - `@Controller('master/school-years')`
    - `GET /` → `@RequirePermissions('school_year.view')`
    - `GET /:id` → `@RequirePermissions('school_year.view')`
    - `POST /` → `@RequirePermissions('school_year.create')` → 201
    - `PATCH /:id` → `@RequirePermissions('school_year.update')`
    - `PATCH /:id/activate` → `@RequirePermissions('school_year.update')`
      Note: route `/:id/activate` harus di-deklarasi SEBELUM `/:id` agar tidak ter-shadow

    **4. Update MasterModule**: tambah SchoolYearsModule ke imports

    Avoid:
    - Jangan lupa urutan route: `/:id/activate` harus SEBELUM `/:id` di controller
    - Transaksi activate harus atomic: gunakan `db.transaction()` dari Drizzle
    - Jangan izinkan deaktivasi manual (PATCH /:id tidak boleh set isActive=false, hanya activate yang boleh)

        INNER JOIN school_units ON classes.school_unit_id = school_units.id

      - ORDER BY school_units.name, classes.level, classes.name
    - `findOne(id)`:
      - SELECT dengan join ke school_units
      - throw NotFoundException jika tidak ada
    - `create(dto)`:
      - Verifikasi schoolUnitId ada: query school_units WHERE id = dto.schoolUnitId
      - Jika tidak ada: throw NotFoundException('Unit sekolah tidak ditemukan')
      - INSERT
    - `update(id, dto)`:
      - findOne dulu
      - Jika dto.schoolUnitId ada: verifikasi school unit exists
      - UPDATE

    - `GET /:id` → `@RequirePermissions('class.view')` → dengan school unit name
    - `POST /` → `@RequirePermissions('class.create')` → 201
    - `PATCH /:id` → `@RequirePermissions('class.update')`

<acceptance_criteria>

## AC-1: Tabel Users
```gherkin
Given user membuka /pengaturan/users
When halaman dimuat
Then tampil tabel dengan kolom: Nama, Email, Role, Status (badge Aktif/Nonaktif), Aksi
And data di-fetch dari GET /users
And loading state ditampilkan saat fetch
```

## AC-2: Create User
```gherkin
Given user klik tombol "Tambah Pengguna"
When modal create terbuka dan form diisi (name, email, password min 6, roleId)
Then POST /users dikirim
And modal tertutup dan tabel refresh
And jika 409 ConflictException, tampil error "Email sudah terdaftar"
```

## AC-3: Edit User
```gherkin
Given user klik tombol Edit pada baris user
When modal edit terbuka dengan data yang sudah terisi
Then PATCH /users/:id dikirim dengan field yang diubah
And modal tertutup dan tabel refresh
```

## AC-4: Deactivate/Activate User
```gherkin
Given user klik toggle Status pada modal edit
When isActive diubah dan form di-submit
Then PATCH /users/:id dengan { isActive: false/true } dikirim
And badge status di tabel terupdate
```

    Zod schema untuk create: { name: min(1), email: email(), password: min(6), roleId: coerce.number().min(1) }
    Zod schema untuk edit (EditFormData): { name: min(1), email: email(), roleId: coerce.number().min(1), isActive: boolean() }
    Gunakan 2 schema terpisah + useForm masing-masing, ATAU satu schema dengan password optional saat edit.

    Fetch:
    - useEffect: GET /users → setUsers
    - useEffect: GET /roles → setRoles (untuk dropdown roleId)

    openCreate(): reset form, setEditing(null), setModalOpen(true)
    openEdit(user): reset form dengan data user (tanpa password), setEditing(user), setModalOpen(true)

    onSubmit:
    - jika !editing: POST /users dengan { name, email, password, roleId }
    - jika editing: PATCH /users/:id dengan { name, email, roleId, isActive } (tanpa password)
    - catch AxiosError: jika status 409 → setError('Email sudah terdaftar'), else 'Gagal menyimpan data.'

    Tabel kolom: Nama, Email, Role, Status, Aksi

    - Status (hanya saat editing): Switch/Select untuk isActive — gunakan native select dengan opsi Aktif/Nonaktif, setValue('isActive', value === 'true')

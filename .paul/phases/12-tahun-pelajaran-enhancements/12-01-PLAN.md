## Goal
Tambah fitur hapus (dengan guard relasi) dan toggle aktif/nonaktif pada halaman Tahun Pelajaran.

Operator perlu bisa menghapus tahun pelajaran yang salah input, dan mengaktifkan tahun pelajaran yang akan digunakan — keduanya belum tersedia di UI meskipun backend activate sudah ada.

## Output
- `DELETE /master/school-years/:id` endpoint baru di backend dengan guard
- Halaman Tahun Pelajaran dengan tombol Hapus + dialog konfirmasi + toggle Status
</objective>

<acceptance_criteria>

## AC-1: Hapus tahun pelajaran tanpa data terkait
```gherkin
Given tahun pelajaran tidak memiliki data di bills, paymentTemplates, atau studentClasses
  AND tahun pelajaran tidak sedang aktif (isActive = false)
When operator klik Hapus → konfirmasi di dialog
Then tahun pelajaran terhapus dari database
  AND list direfresh otomatis
```

## AC-2: Tolak hapus jika ada data terkait atau sedang aktif
```gherkin
Given tahun pelajaran memiliki data terkait (bills/paymentTemplates/studentClasses)
  OR tahun pelajaran sedang aktif (isActive = true)
When operator mencoba hapus dan konfirmasi
Then backend return 400 BadRequestException dengan pesan deskriptif
  AND frontend menampilkan pesan error di UI (tidak crash)
```

## AC-3: Toggle aktifkan tahun pelajaran
```gherkin
Given tahun pelajaran dengan isActive = false
When operator klik badge "Nonaktif" di kolom Status
Then PATCH /master/school-years/:id/activate dipanggil
  AND tahun yang diklik berubah menjadi aktif
  AND tahun yang sebelumnya aktif otomatis menjadi nonaktif
  AND list direfresh
```

## AC-4: Tahun aktif tidak bisa diklik toggle
```gherkin
Given tahun pelajaran dengan isActive = true
When operator melihat kolom Status
Then badge "Aktif" tidak bisa diklik (non-interactive)
```

    Di `school-years.service.ts`, tambah method `remove(id: number)`:
    1. Panggil `findOne(id)` — throw NotFoundException jika tidak ada
    2. Cek `year.isActive === true` → throw `BadRequestException('Tahun pelajaran aktif tidak dapat dihapus')`
    3. Query count relasi secara paralel (Promise.all):
       - `count` dari `bills` where `schoolYearId = id`
       - `count` dari `paymentTemplates` where `schoolYearId = id`
       - `count` dari `studentClasses` where `schoolYearId = id`
    4. Jika total count > 0 → throw `BadRequestException('Tahun pelajaran tidak dapat dihapus karena sudah memiliki data terkait')`
    5. Eksekusi `this.db.delete(schoolYears).where(eq(schoolYears.id, id))`

    Import yang dibutuhkan: `count` dari drizzle-orm, tabel `bills`, `paymentTemplates`, `studentClasses` dari schema index.

    Di `school-years.controller.ts`, tambah endpoint:
    ```
    @Delete(':id')
    @RequirePermissions('school_year.update')
    @HttpCode(HttpStatus.NO_CONTENT)
    remove(@Param('id', ParseIntPipe) id: number) {
      return this.service.remove(id);
    }
    ```
    Import `Delete` dan `HttpStatus` dari @nestjs/common.

    PENTING: Letakkan @Delete(':id') SETELAH @Patch(':id/activate') dan @Patch(':id') agar tidak ada konflik routing.

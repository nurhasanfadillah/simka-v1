<objective>
## Goal
Pindahkan tombol aktivasi tahun pelajaran dari kolom Status ke kolom Aksi. Status hanya jadi badge read-only.

## Purpose
Operator bisa langsung melihat tombol aktivasi di kolom aksi tanpa harus membedakan badge Status yang kadang adalah tombol, kadang badge. Lebih intuitif.

## Output
Kolom Status: badge "Aktif" (hijau) atau "Nonaktif" (abu) — read-only.
Kolom Aksi: tombol Edit, Aktifkan (jika nonaktif), Hapus.
Backend sudah benar — `activate()` deactivate-all-first, tidak bisa ada >1 aktif.
</objective>

<acceptance_criteria>
## AC-1: Tombol aktivasi di kolom Aksi
Given user buka halaman Tahun Pelajaran
When ada tahun nonaktif
Then kolom Aksi menampilkan tombol "Aktifkan" selain Edit dan Hapus

## AC-2: Status column read-only
Given user buka halaman Tahun Pelajaran
When lihat kolom Status
Then tahun aktif: badge hijau "Aktif". Tahun nonaktif: badge abu "Nonaktif". Tidak ada tombol.

## AC-3: Hanya satu tahun aktif
Given user klik "Aktifkan" pada tahun B
When sebelumnya tahun A yang aktif
Then tahun A menjadi nonaktif, tahun B menjadi aktif — dijamin oleh backend `activate()` dalam 1 transaksi.
</acceptance_criteria>

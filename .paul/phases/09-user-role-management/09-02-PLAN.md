<objective>
## Goal
Bangun backend endpoint Roles CRUD (list/create/update) dan PATCH /roles/:id/permissions di dalam AuthModule yang sudah ada.

## Purpose
Super Admin butuh cara membuat role baru dan mengatur permission apa saja yang dimiliki setiap role — tanpa akses langsung ke DB.

## Output
- GET /roles — list semua role + daftar permissionnya
- POST /roles — buat role baru
- PATCH /roles/:id — update nama role
- PATCH /roles/:id/permissions — ganti (replace) daftar permission suatu role secara atomik
</objective>

<acceptance_criteria>

## AC-1: List Roles
```gherkin
Given Super Admin melakukan GET /roles dengan Bearer token
When request diterima
Then response array roles, setiap item berisi: id, name, createdAt, permissions (array { id, code, name })
And dilindungi @RequirePermissions('role.view')
```

## AC-2: Create Role
```gherkin
Given Super Admin POST /roles dengan { name }
When name belum dipakai
Then role baru tersimpan
And response role baru dengan permissions: [] (kosong)
And jika name sudah ada, return 409 ConflictException
And dilindungi @RequirePermissions('role.manage')
```

## AC-3: Update Role
```gherkin
Given Super Admin PATCH /roles/:id dengan { name }
When role dengan id ada
Then nama role terupdate
And response role terupdate
And jika id tidak ada, return 404 NotFoundException
And dilindungi @RequirePermissions('role.manage')
```

## AC-4: Assign Permissions
```gherkin
Given Super Admin PATCH /roles/:id/permissions dengan { permissionIds: number[] }
When role dengan id ada
Then semua rolePermissions lama dihapus dan diganti dengan permissionIds baru (atomic replace)
And response { id, name, permissions: [...] } dengan permissions terbaru
And jika permissionIds kosong [], role jadi tanpa permission
And dilindungi @RequirePermissions('role.manage')
```

    2. Buat `roles.service.ts` — @Injectable(), inject DRIZZLE:
       - `findAll()`: SELECT roles + LEFT JOIN rolePermissions + JOIN permissions
         → return array { id, name, createdAt, permissions: [{ id, code, name }] }
         Gunakan groupBy di aplikasi (bukan SQL) karena Drizzle tidak support GROUP BY dengan array aggregation mudah:
         - Query flat: SELECT roles.id, roles.name, roles.createdAt, permissions.id as permId, permissions.code, permissions.name as permName FROM roles LEFT JOIN role_permissions ON ... LEFT JOIN permissions ON ...
         - Group hasil di TypeScript: Map<roleId, { ...role, permissions: [] }>
         - Return Array.from(map.values())

       - `create(dto: CreateRoleDto)`: cek name unique (ConflictException jika ada), INSERT roles, return { ...role, permissions: [] }

       - `update(id: number, dto: UpdateRoleDto)`: cek role exists (NotFoundException), UPDATE roles.name, return updated role

       - `assignPermissions(id: number, dto: AssignPermissionsDto)`:
         cek role exists (NotFoundException)
         Dalam satu DB transaction:
           1. DELETE FROM role_permissions WHERE role_id = id
           2. Jika permissionIds.length > 0: INSERT INTO role_permissions (role_id, permission_id) VALUES (...)
         Fetch updated role dengan permissions (reuse findOne logic)
         Return { id, name, permissions: [...] }

    1. Buat `roles.controller.ts`:
       ```ts
       @Controller('roles')
       export class RolesController {
         constructor(private rolesService: RolesService) {}

         @Get()
         @RequirePermissions('role.view')
         findAll() { return this.rolesService.findAll() }

         @Post()
         @RequirePermissions('role.manage')
         create(@Body() dto: CreateRoleDto) { return this.rolesService.create(dto) }

         @Patch(':id')
         @RequirePermissions('role.manage')
         update(@Param('id', ParseIntPipe) id: number, @Body() dto: UpdateRoleDto) {
           return this.rolesService.update(id, dto)
         }

         @Patch(':id/permissions')
         @RequirePermissions('role.manage')
         assignPermissions(
           @Param('id', ParseIntPipe) id: number,
           @Body() dto: AssignPermissionsDto,
         ) {
           return this.rolesService.assignPermissions(id, dto)
         }
       }
       ```
       Import: Body, Controller, Get, Param, ParseIntPipe, Patch, Post dari @nestjs/common
       Import: RequirePermissions dari ./decorators/require-permissions.decorator
       Import: CreateRoleDto, UpdateRoleDto, AssignPermissionsDto dari ./dto/role.dto

       PENTING: Route `:id/permissions` harus SEBELUM `:id` agar NestJS routing tidak salah match.
       Urutan dekorator di controller: Patch(':id/permissions') SEBELUM Patch(':id').

<objective>
## Goal
Fix UAT-001: enroll siswa ke kelas lain di tahun ajaran yang sama menyebabkan error generic.

## Purpose
Backend sudah punya endpoint transfer (PATCH /master/student-classes/:id/transfer) untuk pindah kelas.
Frontend selalu memanggil POST (insert baru) tanpa mengecek apakah enrollment di tahun itu sudah ada.
Fix ini membuat frontend deteksi enrollment existing per tahun ajaran dan call endpoint yang tepat.

## Output
- Modal enrollment: "Tambah ke Kelas" → tambah baru jika tahun belum ada enrollment
- Modal enrollment: "Pindah Kelas" → update classId via transfer jika tahun sudah ada enrollment
- Error "Gagal menambahkan enrollment." tidak muncul lagi untuk kasus pindah kelas
</objective>

**Backend endpoints tersedia:**
- POST   /master/student-classes          → enroll baru (insert)
- PATCH  /master/student-classes/:id/transfer  → pindah kelas (update classId)

**Key behavior:**
- Backend punya unique constraint uq_student_school_year pada (studentId, schoolYearId)
- Jika POST ketika sudah ada enrollment di tahun yang sama → 409 Conflict
- Solusi: cek enrollments state untuk schoolYearId yang dipilih → jika ada, call PATCH transfer
